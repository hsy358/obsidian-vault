# 谷雨系统服务器与中间件资源申请清单

整理时间：2026-07-03

## 1. 申请结论

| 场景 | 服务器规格 | 磁盘 | 带宽 | 适用说明 |
| --- | --- | --- | --- | --- |
| 最低可运行 | 8 vCPU / 32 GB | 300 GB SSD | 5 Mbps+ | 测试、联调、小规模验证；文档导入、向量化、ES/Milvus 建库时资源会偏紧 |
| 推荐单机全量 | 16 vCPU / 64 GB | 500 GB SSD | 10 Mbps+ | 推荐申请；适合单机部署 Java、Python、前端、MySQL、Redis、Nacos、ES、Milvus |
| 长期/多人/数据增长 | 应用服务器 8 vCPU / 32 GB + 中间件服务器 16 vCPU / 64 GB | 应用 300 GB SSD + 中间件 1 TB SSD | 10 Mbps+ | 更稳妥；ES、Milvus、MySQL 独立承载，后续扩容更清晰 |

建议操作系统：Ubuntu 22.04 LTS 或 Ubuntu 24.04 LTS。

建议公网：1 个公网 IP 入口即可，公网只开放 80/443，其他端口走内网或安全组白名单。

## 2. 服务器资源清单

| 服务器/资源 | 是否必须 | 推荐规格 | 部署内容 | 备注 |
| --- | --- | --- | --- | --- |
| 应用服务器 | 必须 | 8 vCPU / 32 GB / 300 GB SSD | Nginx、gateway、knowledge/server、im-platform、im-server、guyu_python、preview、前端静态资源 | 如果采用推荐单机全量，可与中间件放同一台 16C64G 服务器 |
| 中间件服务器 | 必须 | 16 vCPU / 64 GB / 500 GB-1 TB SSD | MySQL、Redis、Nacos、Elasticsearch、Milvus、etcd、MinIO | 单机全量时与应用共用；长期环境建议独立 |
| 对象存储资源 | 建议 | 200 GB 起，按文档量扩容 | 上传文件、转换文件、知识库附件 | 工程已有 IOBS 配置；如果不能使用外部 IOBS，需要申请 MinIO/对象存储桶 |
| 备份存储 | 建议 | 200 GB 起 | MySQL 备份、配置备份、关键文件备份 | 建议每日备份，至少保留 7-14 天 |
| GPU/模型服务器 | 可选 | 1 张 GPU / 16 vCPU / 64 GB+ / 500 GB SSD+ | model 服务、本地 embedding/rank/LLM 推理 | 当前申请若使用外部模型 API，不需要 GPU；如果部署 `model` 模块，建议单独申请 |

## 4. 中间件申请清单

| 中间件 | 是否必须 | 建议版本/依据 | 端口 | 建议资源 | 用途 |
| --- | --- | --- | --- | --- | --- |
| MySQL | 必须 | MySQL 8.x，工程使用 `mysql-connector-j 8.3.0`、PyMySQL | 3306 | 2-4 vCPU / 4-8 GB / 100-200 GB SSD 起 | 业务库，当前配置涉及 `appdata`/`cspaicser` 等库 |
| Redis | 必须 | Redis 6.x/7.x | 6379 | 1-2 vCPU / 2-4 GB | 会话、缓存、IM 在线状态/消息相关数据 |
| Nacos | 必须 | Nacos 2.x 更推荐 | 8848，Nacos 2.x 还需 9848/9849 | 1-2 vCPU / 1-2 GB | 服务注册发现、配置中心 |
| Elasticsearch | 必须 | 工程 Java/Python 均有 ES 查询/索引配置 | 9200 | 4 vCPU / 8-16 GB / 100-300 GB SSD | 知识库全文检索、索引查询 |
| Milvus | 必须 | `pymilvus==2.2.8`，建议 Milvus 2.2.x 兼容版本 | 19530、9091 | 4-8 vCPU / 12-24 GB / 200-500 GB SSD | 向量库、知识片段向量检索 |
| etcd | Milvus 依赖 | 与 Milvus 部署配套 | 2379/2380 | 1-2 vCPU / 1-2 GB / 20-50 GB SSD | Milvus 元数据 |
| MinIO | Milvus 依赖/对象存储可复用 | 与 Milvus 部署配套 | 9000/9001 | 2 vCPU / 2-4 GB / 200 GB+ | Milvus 对象数据；也可作为文件对象存储备选 |
| Nginx | 必须 | 稳定版 | 80/443 | 1 vCPU / 512 MB-1 GB | 前端托管、反向代理、HTTPS |
| OnlyOffice | 可选/外部依赖 | 工程已有 OnlyOffice URL 配置 | 80/443 | 若自建建议 4 vCPU / 8 GB | 在线文档编辑；可优先复用外部服务 |

| guyu_python | 8-16 GB | 30-80 GB | 文档解析、向量化、Agent、检索 |
| knowledge/server | 3-6 GB | 20-50 GB | Java 后端 |

## 7. 软件环境申请项

| 类别 | 申请/准备项 |
| --- | --- |
| 基础环境 | Ubuntu 22.04/24.04、时区 Asia/Shanghai、NTP 时间同步 |
| 容器环境 | Docker、Docker Compose 或容器平台等价能力 |
| Java | JDK 8 运行环境；构建机需要 Maven |
| Python | guyu_python 使用 Python 3.9 系列镜像；preview 使用 Python 3.12 系列镜像 |

| Nginx | HTTPS 证书、反向代理、WebSocket upgrade 配置 |

| 配置与密钥 | Nacos 配置、数据库账号、Redis 密码、ES 账号、对象存储凭据、外部平台密钥均需通过环境变量/密钥管理配置，不建议写死在包内 |

## 9. 可直接粘贴的申请描述

用途：谷雨系统全量测试/联调环境，部署 Java 微服务、Python 知识服务、前端静态服务，以及 MySQL、Redis、Nacos、Elasticsearch、Milvus 等中间件。

推荐申请：1 台 `16 vCPU / 64 GB 内存 / 500 GB SSD / Ubuntu 22.04 或 24.04 / 10 Mbps 带宽 / 1 个公网 IP` 的服务器。公网仅开放 80/443，MySQL、Redis、Nacos、ES、Milvus 等端口仅内网白名单访问。

如果需要长期多人使用或数据量增长明显，建议拆分为：应用服务器 `8 vCPU / 32 GB / 300 GB SSD`，中间件服务器 `16 vCPU / 64 GB / 1 TB SSD`。对象存储空间建议 `200 GB` 起，按知识库文件量扩容。

补充说明：当前资源申请默认不在本机部署大模型推理服务，模型能力优先通过外部 API 提供；如需启用工程中的 `model` 服务，需要另行申请 GPU 服务器。
