## 🎯 文档定位（一句话）

> 把"思政课教师 + AI 平台"协同的 8 步流程标准化：从立项到交付，老师只需在 **2 个关键节点**（结构确认 + 内容审核）参与把关，其余由"元知团队 + 平台系统"批量完成。

## 🔑 核心亮点

### 1. 8 步教学端落地闭环（P3）

```
①课程立项 → ②资料提交 → ③资料诊断 → ④知识图谱构建 → 
⑤结构确认 → ⑥资产生成 → ⑦教师审核 → ⑧课程交付
```

- 老师只在 **结构确认（P7）** 和 **教师审核（P9）** 两个节点拍板
- 其余环节是"老师动作 + 元知动作 + 阶段产出"三方协同

### 2. 知识图谱是核心资产（P6）

- **4 类图谱**：知识点 / 价值引领点 / 案例素材 / 模板
- **6 步形成路径**：上传资料 → 自动解析 → 图谱构建 → 抽取知识点/价值引领点 → 建立关系 → 形成图谱
- **底层思想**：课程知识图谱是"先建图谱、再生成、再审核"的基础
- **价值**：思政课程的"价值引领"是独立维度的图谱（区别于一般学科）

### 3. 老师两次关键确认（P7 / P9）

| 节点 | 关注点 | 输出 |
|---|---|---|
| 第一次（结构确认） | 章节顺序/知识点/课程目标/价值引领点映射/口径 | 课程结构图 + 教学流程图 |
| 第二次（内容审核） | 政治方向/理论准确/案例时效/教学可用/格式规范 | 资产包 v1.0/v1.1 |

### 4. 9 类资产批量生成（P8）

PPT 课件 / 教师教案 / 学生讲义 / 章节题库 / 讨论题设计 / 案例素材包 / 视频脚本 / 课程大纲 / 教学计划

**推荐生成顺序**：
```
课程大纲/教学计划 → 教案 → PPT → 讲义 → 题库/讨论题 → 案例素材包 → 视频脚本
```

**3 种生成粒度**：按整门课 / 按章 / 按重点章节

### 5. 5 项验收标准（P14）

1. ✅ 政治方向与价值导向正确
2. ✅ 理论与事实内容准确
3. ✅ 教学可用、结构清晰
4. ✅ 格式规范、符合学校要求
5. ✅ 能够沉淀并支持后续复用更新

### 6. 6-14 天试点周期（P12）

| 阶段 | 时间 | 关键节点 |
|---|---|---|
| 资料准备 | 7 月上—中旬 | 试点课程确定（M1） + 资料齐备（M2）|
| 图谱构建 | 7 月中旬 | 知识图谱通过确认（M3）|
| 资产生成 | 7 月下—8 月上旬 | 资产包完成审核（M4）|
| 课程交付 | 8 月中下旬 | 课程包交付定稿（M5）|

**核心建议**：先 1-2 门课程试点，跑通闭环再复制。

## 🧩 三方协作机制（P13）

| 角色 | 职责 |
|---|---|
| **老师/课程组** | 方向把关、口径确认、关键节点审核、修改意见 |
| **元知团队** | 资料整理、图谱构建、资产组织、迭代优化、交付打包 |
| **平台系统** | 资料解析、知识图谱沉淀、批量生成、版本对比 |

**核心原则**：
- 老师负责方向把关，我们负责组织交付，系统负责高效生成与沉淀
- 口径问题优先由教师确认
- 内容问题由 AI 生成后人工优化
- 格式问题统一按学校模板处理

## 💡 几个值得关注的细节设计

1. **"补充材料不是越多越好，而是越关键、越贴合课程目标越有价值"**（P11）—— 反"形式主义堆料"
2. **"验收通过后统一 v1.0；局部优化则同步新版本记录"**（P14）—— 严格的版本化纪律
3. **"提交越完整，生成越准确；提交越规范，交付越高效"**（P4）—— 把老师侧动作的 ROI 显式化
4. **价值引领点作为独立图谱维度**（P6）—— 思政课程特有设计
5. **7 天快速启动动作**（P15）—— D1 启动 → D7 反馈，D4-D5 集中做知识图谱
