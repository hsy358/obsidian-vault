# 谷雨系统资源申请清单

基于实际代码与部署文件核对后整理。

项目位置：`/root/workspace/guyu`
关键依据：
- 中间件编排：`/root/workspace/guyu/deploy/docker-compose.middleware.yml`
- Gateway：`/root/workspace/guyu/gateway`
- 知识库 Java 服务：`/root/workspace/guyu/knowledge/server`
- IM 服务：`/root/workspace/guyu/im/im-platform`、`/root/workspace/guyu/im/im-server`
- Python 知识服务：`/root/workspace/guyu/guyu_python`

## 一、最低配置（可运行下限）

适用场景：
- 测试环境
- 功能联调
- 小规模验证
- 允许资源余量较小，不追求高并发

推荐清单：
- CPU：8 vCPU
- 内存：32 GB
- 磁盘：300 GB SSD
- 带宽：5 Mbps 以上
- 公网 IP：1 个
- 操作系统：Ubuntu 24.04 LTS
- swap：8–16 GB

说明：
- 这是“能跑全量服务”的最低建议，不是舒适配置
- 在文档导入、向量化、Milvus 建库、ES 建索引时，内存压力会比较明显
- 如果多人同时使用，稳定性和响应速度会下降

## 二、最合适配置（推荐申请）

适用场景：
- 单机全量部署
- 长时间稳定运行
- 测试、演示、联调一体
- 有一定导入、检索、向量处理压力

推荐清单：
- CPU：16 vCPU
- 内存：64 GB
- 磁盘：500 GB SSD
- 带宽：10 Mbps 以上
- 公网 IP：1 个
- 操作系统：Ubuntu 24.04 LTS
- swap：16 GB

说明：
- 这是目前最适合的单机配置
- 能较稳地承载 MySQL、Redis、Nacos、Elasticsearch、Milvus，以及 Java/Python/前端服务
- 适合作为谷雨系统单机全量测试环境

## 三、为什么需要这个配置

系统并不是单一知识库服务，而是完整多组件系统，实际包含：
- MySQL
- Redis
- Nacos
- Elasticsearch
- Milvus（含 etcd、MinIO）
- Gateway
- knowledge/server
- im-platform
- im-server
- guyu_python
- admin_web
- serv_web

其中最吃资源的部分是：
- Elasticsearch
- Milvus
- guyu_python
- knowledge/server

## 四、资源大致分配参考

按推荐配置估算：
- MySQL：4 GB
- Redis：1 GB
- Nacos：1 GB
- Elasticsearch：8 GB
- Milvus + etcd + MinIO：10–12 GB
- guyu_python：8–12 GB
- knowledge/server：3–4 GB
- im-platform：2–4 GB
- im-server：1–2 GB
- gateway + nginx + 前端 + 系统余量：10–14 GB

## 五、网络与端口建议

建议开放：
- 80/443：前端与反向代理
- 3306：MySQL（建议仅内网或白名单）
- 6379：Redis（建议仅内网或白名单）
- 8848：Nacos
- 9200：Elasticsearch（建议仅内网或白名单）
- 19530：Milvus（建议仅内网或白名单）
- 9000/9001：MinIO（建议仅内网）
- 9091：Milvus 监控/管理相关端口（建议仅内网）
- 8080：gateway（通常走 nginx 反代）
- 8089：knowledge/server（可仅内网）
- 8667：guyu_python（可仅内网）
- 8888：im-platform（可仅内网）

## 六、申请时可直接使用的文字

用途：
谷雨系统单机全量测试/联调环境。

部署内容：
Java 微服务、Python 知识服务、前端服务，以及 MySQL、Redis、Nacos、Elasticsearch、Milvus 等中间件。

最低配置：
8核 CPU / 32GB 内存 / 300GB SSD / Ubuntu 24.04 / 5Mbps 带宽 / 1个公网IP。

推荐配置：
16核 CPU / 64GB 内存 / 500GB SSD / Ubuntu 24.04 / 10Mbps 带宽 / 1个公网IP。

补充说明：
模型能力优先通过外部 API 提供，不在本机部署大模型推理服务；当前服务器主要承载业务服务、知识检索、向量库及中间件。

## 七、结论

如果你只想先把系统跑起来，申请：
- 8 vCPU / 32 GB / 300 GB SSD

如果你想减少后续扩容和卡顿，建议直接申请：
- 16 vCPU / 64 GB / 500 GB SSD
