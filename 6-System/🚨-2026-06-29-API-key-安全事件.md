## 教训（写入 MEMORY.md）

1. **任何含凭据/密钥的 .json 都必须加 .gitignore**
   - 已经在 `.obsidian/plugins/*/data.json` 加 ignore
   - 但**默认应该 ignore**所有 `**/data.json`，按需 white-list

2. **AI Agent 处理 .obsidian 目录时要特别小心**
   - 之前 Para 重构时没注意到 claudian/data.json 含 API key
   - 教训：扫描所有 data.json 是否含 `apiKey` / `key` / `token` / `password` 字段

3. **Git history 是永久的**
   - 一旦 commit 到公开仓库，**就是永久公开**
   - 任何 .gitignore 都是"防未来"，不防历史
   - **预防 > 治疗**

4. **新插件安装 SOP 应加安全审计**
   - 装任何 Obsidian 插件后：
     - 检查 `data.json` 内容
     - 如果含凭据 → 立即加 `.gitignore` 规则
     - 不在 commit
