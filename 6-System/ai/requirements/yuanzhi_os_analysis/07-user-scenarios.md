##详细场景

### S-01：教务处上传培养方案 PDF

**角色**：教务处管理员（R02）
**空间**：管理空间
**前置条件**：已登录系统，拥有资源上传权限

**流程**：
1. 教务处管理员进入管理空间，点击"上传培养方案"
2. 选择本地 PDF 文件（大小 ≤ 100MB）
3. 系统显示上传进度，上传完成后自动触发 AI 解析
4. ResourceExtractService 提取 PDF 文本并解析结构化指标
5. 解析结果展示（课程名称列表、学时、学分、考核方式）
6. 管理员确认指标无误后，保存至培养方案记录
7. 若 PDF 为图片型，则先经 OCR 识别再解析

**AI 依赖**：`resource.pdf-extract`, `resource.ocr`
**触发入口**：`GlobalDock.vue` → "上传资源" 或 `ResourceController.java` → `/api/resource/upload`
**数据影响**：写入 `resource` 表，`program_plan` 表更新
**通知**：解析完成后系统提示；失败则提示错误原因
**异常**：
- 文件格式不支持 → 提示"仅支持 PDF/DOC/PPT/图片"
- PDF 加密 → 提示"该 PDF 已加密，请先解密后上传"
- 解析失败 → 提示"解析失败，可手动编辑指标"

### S-02：专业带头人发起长文裂变

**角色**：专业带头人（R04）
**空间**：专业空间
**前置条件**：已选择具体专业，专业状态为 published 或 first-reviewed

**流程**：
1. 专业带头人进入专业空间，选择具体专业
2. 点击"AI 辅助"→"长文裂变任务"
3. AISidebar 打开，用户粘贴或上传长文（支持 PDF/纯文本）
4. 系统调用 AiChatService.splitMission() 启动裂变
5. AI 返回子任务列表（可编辑、可删除、可新增）
6. 用户确认子任务列表，保存至专业建设任务
7. 子任务可分配给相关教师执行

**AI 依赖**：`ai.doc-split`
**触发入口**：`AISidebar.vue` → 长文裂变入口，或 `GlobalDock.vue`
**数据影响**：新建 `ai_mission` 表记录，任务结果写入 `mission_result` 表
**通知**：任务完成后 AISidebar 通知；失败则显示错误
**异常**：
- 文档超长（> 50,000字）→ 自动分段处理，结果合并后展示
- AI 置信度 < 0.85 → 标记"需人工确认"，高亮低置信度任务
- 网络超时 → mission 状态=failed，可一键重试

### S-03：教师上传扫描课件进行 OCR

**角色**：一线教师（R05）
**空间**：教师空间
**前置条件**：已登录，拥有资源上传权限

**流程**：
1. 教师进入教师空间，点击"上传课件"
2. 选择图片或扫描件 PDF（JPG/PNG/PDF 图片型）
3. 系统自动触发 OCR 识别（ResourceExtractService）
4. 识别结果以可编辑文本形式展示
5. 教师确认后，文本保存为教学资源
6.资源自动关联当前课程

**AI 依赖**：`resource.ocr`
**触发入口**：`GlobalDock.vue` 或资源上传页
**数据影响**：OCR 结果写入 `resource_text` 表，关联课程 ID
**异常**：
- 图片模糊 → 提示"图片质量过低，请上传更清晰的图片"
- 手写体 → 提示"当前版本仅支持印刷体识别，手写体可能不准确"

### S-04：专业带头人构建课程知识图谱

**角色**：专业带头人（R04）
**空间**：专业空间
**前置条件**：专业已发布，课程体系已完成设计

**流程**：
1. 专业带头人进入专业空间，点击"知识图谱构建"
2. KnowledgeGraphView.vue 展示已有图谱（无则空图）
3. 选择课程体系，系统调用 GraphITI 引擎分析课程关联
4. AI 自动生成课程节点和关系边
5. 专业带头人可手动增删节点、编辑关系
6. 提交评审，节点状态变为 `ai-generated → under-review`
7. 评审专家确认后，节点状态变为 `published`

**AI 依赖**：`kg.engine`, `kg.frontend`
**触发入口**：`KnowledgeGraphView.vue`
**数据影响**：节点写入 FalkorDB/Neo4j，向量嵌入写入 PgVector
**异常**：
- 图谱构建超时 → 提示"图谱构建超时，请减少课程数量后重试"
- 节点冲突 → 提示"该课程节点已存在，请选择合并或覆盖"

---

### S-05：评审专家确认 AI 生成里程碑

**角色**：评审专家（R06）
**空间**：教师空间
**前置条件**：有待评审的 AI 生成里程碑（来自摘要任务）

**流程**：
1. 评审专家登录，进入"待评审"列表
2. 点击具体里程碑任务，查看 AI生成的摘要和里程碑节点
3. 对每个里程碑节点：确认 / 驳回（需填写理由）
4. 确认后节点状态变为 `published` 并通知专业带头人
5. 驳回后节点退回 `rejected`，专业带头人修订后重新提交

**AI 依赖**：`ai.summary`
**触发入口**：AISidebar 通知 / 评审列表页
**数据影响**：里程碑节点状态更新，写入 `milestone_review_log` 表
**异常**：
- 评审超时（7天未处理）→ 自动发送催办通知
- 批量驳回 → 需填写统一的驳回理由

---

### S-06：系部主任提交专业方案审核

**角色**：系部主任（R03）
**空间**：专业空间
**前置条件**：专业方案已完成编辑，状态为 `drafting`

**流程**：
1. 系部主任进入专业空间，选择具体专业
2. 点击"提交审核"，表单自动校验必填项
3. 确认提交，专业状态从 `drafting` 变为 `submitted`
4. 系统通知教务处管理员有新的待审核方案
5. 进入初审流程（见09-process-flows.md）

**触发入口**：`ProgramPlanView.vue` → "提交审核"按钮
**数据影响**：`professional_project` 表状态更新
**异常**：
- 表单校验不通过 → 显示具体缺失字段，不提交
- 专业带头人未完成课程体系设计 → 提示"请先完成课程体系设计再提交"

### S-07：校领导查看数据看板

**角色**：校领导（R01）
**空间**：管理空间
**前置条件**：已登录，角色为校领导

**流程**：
1. 校领导登录，默认进入首页 HomeView.vue
2. 首页展示数据看板（专业总数、建设进度、AI 使用统计等）
3. 可点击具体指标跳转至详情页
4. 可筛选时间范围（近一年/近一月/本周）

**触发入口**：`HomeView.vue` → 数据看板面板
**数据影响**：只读，无写入
**异常**：
- 数据加载失败 → 显示骨架屏 + "数据加载失败，点击重试"
- 无数据 → 显示空态"暂无数据"

---

### S-08：系统管理员配置角色权限

**角色**：系统管理员（R07）
**空间**：全局
**前置条件**：已登录，角色为系统管理员

**流程**：
1. 系统管理员进入"系统管理"→"权限配置"
2. 展示当前角色列表和权限矩阵
3. 可新增角色（需指定角色名、描述）
4. 可修改角色权限（勾选/取消权限项）
5. 保存时系统记录变更日志（审计日志）

**触发入口**：系统管理菜单 →权限配置页
**数据影响**：写入 `role` 表、`permission` 表、`audit_log` 表
**异常**：
- 权限冲突（如删除管理员自身权限）→ 阻止并提示"不能修改自己的管理员权限"
- 修改前已有用户分配该角色 → 提示"修改将影响 N 个已有用户"

---

### S-09：AI 辅助教师编写教案

**角色**：一线教师（R05）
**空间**：教师空间
**前置条件**：已登录，课程已关联

**流程**：
1. 教师在教案编辑页面，点击"AI 辅助"按钮
2. AISidebar 打开，教师输入问题或选择预设模板
3. AiChatService 处理请求，返回 AI 生成内容
4. 教师可将 AI 内容插入教案编辑器
5. AI记录本次交互至对话历史

**AI 依赖**：`ai.chat`
**触发入口**：`AISidebar.vue` 或教案编辑器内"AI 辅助"按钮
**数据影响**：对话历史写入 `ai_chat_history` 表
**异常**：
- AI 响应超时（> 30s）→ 显示"AI响应超时，请重试"
- 内容违规 → 拒绝输出并提示"内容包含不合规信息"

---

### S-10：教务处管理员配置专业建设参数

**角色**：教务处管理员（R02）
**空间**：管理空间
**前置条件**：已登录

**流程**：
1. 教务处管理员进入"专业建设"→"参数配置"
2. 可配置：专业申报周期、审核流程、评审专家池
3. 保存后自动生效

**触发入口**：管理空间 → 专业建设配置页
**数据影响**：写入 `system_config` 表
**异常**：
- 参数变更影响已有流程 → 提示"当前配置变更会影响 N 个进行中的专业，确认继续？"
