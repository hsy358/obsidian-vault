### UC-001：上传教学资源

**参与角色**：一线教师（R05）、教务处管理员（R02）
**目标**：将教学资源（PDF/Word/PPT/图片）上传至系统，供后续 AI 解析使用
**前置条件**：用户已登录，且资源上传权限已分配

**主流程**：
1. 用户进入资源上传页面（GlobalDock → 上传资源 或资源页）
2. 用户点击"选择文件"，选择本地文件
3. 系统校验文件格式（PDF/DOC/DOCX/PPT/PPTX/JPG/PNG）和大小（≤ 100MB）
4. 校验通过，上传文件至服务器，存入存储服务
5. 系统返回上传成功，显示文件卡片（含文件名、大小、上传时间）
6. 触发 AI 解析流程（见 UC-002 或 UC-003）

**分支流程**：
- 分支A1：文件格式不支持 → 显示错误提示"不支持该格式，请上传 PDF/DOC/PPT/图片"，流程终止
- 分支A2：文件大小超限（> 100MB）→ 显示"文件超过 100MB，请压缩后上传"，流程终止
- 分支A3：文件已存在（MD5 重复）→ 提示"该文件已上传，是否覆盖？"（覆盖/取消）

**异常流程**：
- 异常E1：上传中断（网络断开）→ 显示"上传中断，点击重试"，已上传部分自动清理
- 异常E2：服务器存储失败 → 显示"上传失败，请稍后重试"，提示错误码供排查
- 异常E3：PDF 加密 → 显示"该 PDF 已加密，请先解密后上传"

**数据影响**：
- 写入 `resource` 表（resource_id, file_name, file_path, file_size, file_type, uploaded_by, uploaded_at, status）
- 写入文件存储（对象存储/NFS）

**通知/日志**：
-成功：系统通知"上传成功，AI 解析中..."
- 失败：记录 error_log

**可复用能力**：`resource.upload`（CAPABILITY-GRAPH.yaml）
**阻塞问题**：
- Q001: 文件大小 100MB 上限是否可调整？影响存储服务容量规划
- Q002: 文件 MD5 去重策略是否需要支持多版本？

### UC-002：PDF 解析提取指标

**参与角色**：系统（自动触发）
**目标**：从 PDF 文档中自动提取结构化指标（课程名、学时、学分、考核方式）
**前置条件**：资源上传完成，文件类型为 PDF（文本型，非扫描件）

**主流程**：
1. 系统自动触发，或用户手动点击"AI 解析"
2. ResourceExtractService 读取 PDF 文件流
3. PDF 解析引擎提取文本内容和布局信息
4. AI 模型分析文本，识别结构化指标（课程名、学时、学分、考核方式）
5. 解析结果写入 `resource_extracted` 表
6. 前端展示解析结果卡片
7. 用户确认指标正确性，可手动修正

**分支流程**：
- 分支B1：PDF 为图片型（扫描件）→ 触发 UC-003（OCR识别），再继续解析
- 分支B2：PDF 加密 → 标记解析失败，通知上传用户
- 分支B3：PDF 为空或无有效文字 → 显示"PDF 内容为空，请检查文件后重新上传"

**异常流程**：
- 异常E1：解析超时（> 60s）→ 标记任务 failed，通知用户可重试
- 异常E2：解析结果字段缺失率 > 50% → 标记"需人工补录"，提示用户补充
- 异常E3：PDF 版面复杂（多栏/表格）→ 解析质量下降，提示"复杂版面可能影响准确性"

**数据影响**：
- 写入 `resource_extracted` 表（指标字段：course_name, hours, credits, assessment_type, confidence_score）
- 更新 `resource` 表 status = 'extracted'

**通知/日志**：
- 解析完成：AISidebar 通知用户"文档解析完成，请确认指标"
- 解析失败：通知用户并提供错误原因

**可复用能力**：`resource.pdf-extract`
**阻塞问题**：
- Q003: 哪些指标字段是必须提取的？字段清单需要产品确认
- Q004: 不同院校的培养方案模板不同，解析准确率如何保障？

### UC-003：多模态 OCR 识别

**参与角色**：系统（自动触发）
**目标**：对图片或扫描件 PDF 进行文字识别
**前置条件**：资源为 JPG/PNG，或 PDF 为图片型（扫描件）

**主流程**：
1. ResourceExtractService 检测到图片型资源
2. OCR 引擎对图片进行文字识别（印刷体优先）
3. 识别结果以可编辑文本形式返回
4. 用户确认识别结果，可手动修正
5. 修正后文本保存为教学资源

**分支流程**：
- 分支C1：图片模糊 → 显示"图片质量过低，请上传更清晰的图片"，不保存
- 分支C2：包含手写体 → 提示"当前版本仅支持印刷体，手写体识别可能不准确"，允许用户确认后继续

**异常流程**：
- 异常E1：图片尺寸过大（> 50MB）→ 自动压缩后识别
- 异常E2：非文字内容为主（如流程图）→ 提示"未检测到文字内容，请确认文件"

**可复用能力**：`resource.ocr`
**阻塞问题**：
- Q005: OCR 识别准确率 95% 的评估标准是什么？是否需要人工抽检？

### UC-004：发起长文裂变任务

**参与角色**：专业带头人（R04）
**目标**：将长文档拆解为可执行的子任务列表
**前置条件**：专业已创建，用户已选择具体专业

**主流程**：
1. 专业带头人进入 AISidebar，选择"长文裂变"
2. 用户粘贴长文本或上传文档
3. AiChatService.splitMission() 接收输入
4. AI 分析文档结构，返回子任务列表（JSON格式，含任务标题、描述、预计时长）
5. 子任务列表展示，可编辑/删除/新增任务
6. 用户确认，保存任务列表至 `ai_mission` 表
7. 任务可分配给课程相关教师

**分支流程**：
- 分支D1：文档字数 ≤ 5000 → 直接裂变
- 分支D2：文档字数 > 5000 → 分段裂变后合并
- 分支D3：AI 置信度 < 0.85 → 低置信度任务高亮标记，用户确认后保存

**异常流程**：
- 异常E1：任务失败 → mission 状态=failed，显示"裂变失败，请重试"
- 异常E2：裂变结果为空 → 提示"未识别到有效任务，请检查文档内容"
- 异常E3：超时（> 120s）→ 标记超时，可一键重试

**数据影响**：
- 新建 `ai_mission` 表记录（mission_id, mission_type='doc_split', input_text, result_json, status, created_by, created_at）
- 任务结果写入 `mission_result` 表

**通知/日志**：
- 完成：AISidebar 通知"裂变完成，共生成 N 个子任务"
- 失败：显示错误原因，记录 error_log

**可复用能力**：`ai.doc-split`
**阻塞问题**：
- Q006: 裂变任务的最大输入字数限制是多少？超出如何分段？
- Q007: 子任务是否需要支持指派给特定教师？

### UC-005：生成文档摘要与里程碑

**参与角色**：专业带头人（R04）
**目标**：对长文档生成摘要和关键里程碑节点
**前置条件**：资源已解析（PDF 或 OCR）

**主流程**：
1. 专业带头人选择已解析的文档，点击"生成摘要"
2. AiChatService.generateSummary() 处理文档内容
3. AI 返回摘要文字（≤ 5000字）和里程碑节点列表（JSON）
4. 摘要展示于文档详情页；里程碑节点展示为时间轴
5. 用户确认里程碑，可编辑/删除/新增
6. 提交评审，里程碑状态变为 `under-review`

**分支流程**：
- 分支E1：AI 置信度 ≥ 0.95 → 里程碑节点自动标记为 `published`（无需评审）
- 分支E2：置信度 < 0.95 → 需评审专家确认

**异常流程**：
- 异常E1：文档内容为空 → 提示"文档内容为空，无法生成摘要"
- 异常E2：摘要生成超时 → 标记失败，允许重试

**数据影响**：
- 摘要写入 `document_summary` 表
- 里程碑节点写入 `milestone` 表（状态 `ai-generated` 或 `under-review`）

**可复用能力**：`ai.summary`
**阻塞问题**：
- Q008: AI 生成结果的置信度阈值（0.95 自动通过）是否可以配置？

### UC-006：构建课程知识图谱

**参与角色**：专业带头人（R04）
**目标**：基于课程内容构建知识网络，支持关联分析与教学路径推荐
**前置条件**：专业已发布，课程体系已完成

**主流程**：
1. 专业带头人进入 KnowledgeGraphView.vue，点击"构建图谱"
2. 选择关联的课程列表（TreeSelectWidget.vue）
3. 系统调用 GraphITI 引擎，传入课程内容文本
4. GraphITI 生成课程节点和关系边（JSON）
5. KnowledgeGraphView.vue 可视化展示图谱
6. 专业带头人可手动增删节点、编辑关系
7. 提交评审，节点状态变为 `ai-generated → under-review`

**分支流程**：
- 分支F1：已有图谱 → 提示"是否覆盖现有图谱或增量追加？"
- 分支F2：课程数量 > 50 → 提示"课程数量较多，构建可能耗时较长，确认继续？"

**异常流程**：
- 异常E1：图谱构建超时（> 180s）→ 标记失败，显示"图谱构建超时，请减少课程数量"
- 异常E2：节点关系冲突 → 高亮冲突节点，提示用户手动解决

**数据影响**：
- 节点写入 FalkorDB/Neo4j（`knowledge_node`）
- 边写入 FalkorDB/Neo4j（`knowledge_edge`）
- 向量嵌入写入 PgVector

**可复用能力**：`kg.engine`, `kg.frontend`
**阻塞问题**：
- Q009: GraphITI引擎的调用方式是 REST API 还是 SDK？集成方式待确认
- Q010: 图数据库选型（FalkorDB vs Neo4j）是否已确定？

### UC-007：评审确认 AI 图谱节点

**参与角色**：评审专家（R06）
**目标**：审核 AI 生成的图谱节点，确保准确性后发布
**前置条件**：有待评审的图谱节点（状态=`under-review`）

**主流程**：
1. 评审专家进入"待评审"列表，看到 AI 生成的图谱节点
2. 点击节点查看详情（课程名、关联课程、关系类型、置信度）
3. 对每个节点：点击"确认"或"驳回"（驳回需填写理由）
4. 确认后节点状态变为 `published`
5. 驳回后节点退回 `rejected`，通知专业带头人修订

**分支流程**：
- 分支G1：批量评审 → 勾选多个节点，一键确认/驳回
- 分支G2：评审超时（7天）→ 自动发送催办通知给评审专家

**异常流程**：
- 异常E1：评审过程中节点被其他用户编辑 → 提示"节点已被修改，请刷新后重审"

**数据影响**：
-节点状态更新为 `published` 或 `rejected`
- 评审记录写入 `milestone_review_log` 表
- 通知写入 `notification` 表

**可复用能力**：`kg.engine`
**阻塞问题**：
- Q011: 评审专家是否可以批量评审？批量驳回时是否必须填写统一理由？

### UC-008：提交专业方案审核

**参与角色**：系部主任（R03）
**目标**：将专业方案从草稿状态提交至初审
**前置条件**：专业方案已完成编辑，状态=`drafting`

**主流程**：
1. 系部主任在 ProgramPlanView.vue 中点击"提交审核"
2. 系统自动校验必填项（课程体系完整、指标完整等）
3. 校验通过，专业状态从 `drafting` → `submitted`
4. 系统通知教务处管理员"有待审核的专业方案"
5. 进入初审流程（见 UC-009）

**分支流程**：
- 分支H1：校验不通过 → 显示具体缺失字段，高亮提示，不提交
- 分支H2：课程体系未完成设计 → 提示"请先完成课程体系设计再提交"

**异常流程**：
- 异常E1：提交时网络断开 → 提示"提交失败，请检查网络后重试"，状态保持 `drafting`

**数据影响**：`professional_project` 表 status = `submitted`，created_at/updated_at 更新

**通知/日志**：
- 成功：通知教务处管理员；系部主任收到"已提交"提示
- 失败：系部主任收到错误提示

**可复用能力**：`plan.frontend`

### UC-009：初审专业方案

**参与角色**：教务处管理员（R02）
**目标**：对提交的专业方案进行初审，通过或驳回
**前置条件**：有状态=`submitted`的专业方案

**主流程**：
1. 教务处管理员进入"待审核"列表
2. 点击具体方案，查看方案详情（系部主任提交的内容）
3. 填写初审意见（通过/驳回）
4. 若通过：状态从 `submitted` → `first-reviewed`，通知评审专家
5. 若驳回：状态回退 `submitted` → `drafting`，通知系部主任并附带驳回理由

**分支流程**：
- 分支I1：需要补充材料 → 驳回并留言，系部主任补充后重新提交

**异常流程**：
- 异常E1：教务处管理员无权限审核该专业 → 提示"您没有该专业的审核权限"

**数据影响**：
- `professional_project` 表 status 更新
- 审核记录写入 `review_log` 表（reviewer_id, review_type='first', opinion, created_at）

**通知/日志**：通知系部主任审核结果

**阻塞问题**：
- Q012: 初审和终审是否可以由同一角色担任？还是必须分开？

### UC-010：终审专业方案

**参与角色**：评审专家（R06）
**目标**：对初审通过的专业方案进行终审，通过则发布
**前置条件**：专业方案状态=`first-reviewed`

**主流程**：
1. 评审专家进入"终审"列表
2. 查看方案详情和初审意见
3. 填写终审意见（通过/驳回）
4. 若通过：状态 → `final-reviewed` → `published`，通知系部主任和专业带头人
5. 若驳回：状态回退 → `drafting`，通知相关人员

**异常流程**：
- 异常E1：评审专家与系部主任为同一用户 → 系统应阻止（不能自己审自己的方案）

**数据影响**：`professional_project` 表 status 更新，`review_log` 写入终审记录

**可复用能力**：`plan.frontend`

4. 可切换时间范围（近一年/近一月/本周）

**分支流程**：
- 分支J1：数据加载中 → 显示骨架屏
- 分支J2：加载失败 → 显示"加载失败，点击重试"

**异常流程**：
- 异常E1：无数据 → 显示空态"暂无数据"

**数据影响**：只读，无数据写入

### UC-012：AI 辅助编写教案

**参与角色**：一线教师（R05）
**目标**：在编写教案时获取 AI 辅助（生成大纲、推荐内容等）
**前置条件**：教师已登录，课程已关联

**主流程**：
1. 教师在教案编辑器中点击"AI 辅助"
2. AISidebar 打开，教师输入问题或选择预设模板（如"生成课程大纲"）
3. AiChatService 处理请求，返回 AI 生成内容
4. 教师可将 AI 内容插入编辑器
5. 交互历史保存至 `ai_chat_history`

**异常流程**：
- 异常E1：AI响应超时（> 30s）→ 显示"AI响应超时，请重试"
- 异常E2：内容包含不合规信息 → 拒绝输出并提示

**可复用能力**：`ai.chat`

### UC-013：配置角色权限

**参与角色**：系统管理员（R07）
**目标**：配置系统角色和权限矩阵
**前置条件**：用户为系统管理员

**主流程**：
1. 管理员进入"系统管理"→"权限配置"
2. 查看当前角色列表（支持新增/编辑/删除角色）
3. 点击角色，查看/编辑该角色的权限项
4. 保存时记录审计日志
5. 权限变更实时生效（已登录用户需重新登录）

**异常流程**：
- 异常E1：删除管理员自身权限 → 阻止操作并提示
- 异常E2：修改影响已有用户 → 提示影响的用户数量，确认后继续

**数据影响**：`role` 表、`permission` 表更新，审计日志写入 `audit_log`

### UC-014：空间切换

**参与角色**：全体角色（多空间角色）
**目标**：在三大空间之间切换，保留上下文
**前置条件**：用户角色同时归属多个空间

**主流程**：
1. 用户在顶部导航栏点击空间切换器
2. 选择目标空间（如从"管理空间"切换至"专业空间"）
3. 路由跳转到目标空间首页，菜单刷新
4. 状态管理（store）切换，保留当前操作的上下文（如编辑中内容）

**分支流程**：
- 分支K1：当前有未保存内容 → 提示"切换空间将丢失未保存内容，是否继续？"
- 分支K2：目标空间无权限 → 提示"您没有该空间的访问权限"，不切换

**异常流程**：
- 异常E1：切换后页面空白 → 提示"页面加载异常，请刷新重试"

**数据影响**：路由跳转，store 状态切换

**阻塞问题**：
- Q013: 空间切换时未保存内容的处理策略是什么？自动保存草稿还是提示丢弃？
