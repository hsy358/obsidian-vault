# 02-clarification.md — Yuanzhi OS 业务需求澄清

> 基于原始需求文档进行业务需求结构化澄清。

---

## 1. 需求摘要

Yuanzhi OS 是一个 AI 原生教学操作系统，通过三大空间（管理空间、专业空间、教师空间）为六类角色提供专业建设、人才培养方案管理、一线教学的全流程支撑。系统深度集成 AI 能力（PDF解析、长文裂变、摘要里程碑、多模态OCR、课程知识图谱），目标是全面替代传统 SDS 教学管理系统。

---

## 2. 目标与非目标

### 2.1 目标

- 构建 AI 原生的教学操作系统，覆盖教学管理全生命周期
- 实现三大空间的角色化、权限隔离的访问控制
- 通过 AI 能力大幅降低教学文档处理、知识管理的人工成本
- 全面替代 SDS 系统并完成数据迁移
- 基于知识图谱实现课程体系的结构化、可视化、可追踪管理

### 2.2 非目标

- 不支持学生端功能（仅面向教学管理者和教师）
- 不替代现有的教务排课系统（仅做数据集成）
- 不提供面向家长的公示功能
- 初期不开放 API 给第三方系统集成

## 5. 原子需求

### 5.1 空间与导航

- **AR-001**：系统根据用户角色自动展示对应空间的入口
- **AR-002**：用户可在三大空间之间切换，空间状态独立
- **AR-003**：首页 HomeView.vue 根据角色展示个性化面板

### 5.2 教学资源管理

- **AR-010**：用户可上传 PDF/Word/PPT/图片格式的教学资源
- **AR-011**：系统对上传资源进行 AI 解析，提取结构化指标
- **AR-012**：ResourceController.java 提供资源上传、解析、查询接口
- **AR-013**：ResourceExtractService.java 完成 PDF 文本提取与 OCR 识别
- **AR-014**：多模态 OCR 支持印刷体和基本手写体识别

### 5.3 AI 核心能力

- **AR-020**：AI 对话服务（AiChatService.java）提供多轮对话能力
- **AR-021**：长文裂变：输入长文档，输出可执行子任务列表
- **AR-022**：文档摘要：为文档生成摘要文字 + 里程碑节点列表
- **AR-023**：AI 侧边栏（UISidebar.vue）提供统一的 AI 交互入口
- **AR-024**：GlobalDock.vue 提供 AI 能力的全局快捷入口

### 5.4 人才培养方案

- **AR-030**：支持培养方案 Excel/Word 格式导入（PlanImportParser.java）
- **AR-031**：ProgramPlanView.vue 展示/编辑培养方案
- **AR-032**：TreeSelectWidget.vue 支持专业/课程/章节多级选择
- **AR-033**：培养方案与知识图谱联动（课程节点自动关联）

### 5.5 知识图谱

- **AR-040**：KnowledgeGraphView.vue 可视化展示课程知识网络
- **AR-041**：GraphITI 引擎支持知识图谱节点/边的 CRUD
- **AR-042**：知识图谱数据存储于 FalkorDB/Neo4j，向量检索依赖 PgVector
- **AR-043**：专业带头人可构建和编辑课程知识图谱
- **AR-044**：AI 生成的图谱节点需经评审专家确认后才能发布

### 5.6 系统管理

- **AR-050**：系统管理员可配置用户角色和权限
- **AR-051**：系统保留关键操作审计日志
- **AR-052**：支持数据导出（Excel/JSON）

## 6. 主流程

### 6.1 专业建设主流程

```
[系部主任/专业带头人创建专业规划]
    → [教务处管理员初审]
    → [评审专家终审]
    → [专业带头人设计课程体系]
    → [AI辅助构建知识图谱]
    → [一线教师使用 AI 辅助编写教案]
    → [校领导数据看板查看建设成效]
```

### 6.2 AI 文档处理流程

```
[用户上传教学文档（PDF/图片）]
    → [ResourceController接收并存储]
    → [ResourceExtractService 解析（PDF文本提取 或 OCR识别）]
    → [AiChatService 执行下游 AI 任务：裂变/摘要/图谱构建]
    → [结果写入 PostgreSQL（结构化数据）和 FalkorDB/Neo4j（图谱数据）]
    → [前端展示（KnowledgeGraphView / ProgramPlanView）]
```

## 7. 分支流程

### 7.1 资源上传分支

```
[上传资源]
    ├── 普通格式（PDF文本型）→ 直接 PDF 解析提取
    ├── 图片型 PDF / 扫描件 → OCR 识别后再解析
    └── 格式不支持 → 提示"格式不支持，请上传 PDF/Word/PPT/图片"
```

### 7.2 长文裂变分支

```
[发起长文裂变任务]
    ├── 文档 ≤ 5000字 → 直接裂变
    ├── 文档 > 5000字 → 分段裂变后合并
    └── AI 置信度 < 阈值 → 标记"需人工确认"
```

### 7.3 知识图谱节点评审分支

```
[AI 生成图谱节点]
    ├── 评审专家确认 → 发布上线
    ├── 评审专家驳回 → 专业带头人修订后重新提交
    └── 超过评审时限（7天）→ 自动发送催办通知
```

## 8. 对象状态流转

### 8.1 专业（ProfessionalProject）

| 状态 | 说明 | 可执行操作 | 触发条件 |
|---|---|---|---|
| drafting | 草稿（系部主任创建） | 编辑、提交 | 创建时 |
| submitted | 已提交待初审 | 教务处管理员初审 | 提交时 |
| first-reviewed | 初审通过 | 评审专家终审 | 初审通过 |
| final-reviewed | 终审通过 | 建设中 | 终审通过 |
| published | 已发布 | 编辑（需重新审核） | 发布时 |
| archived | 已归档 | 只读 | 归档操作 |

### 8.2 AI 任务（AIMission）

| 状态 | 说明 | 可执行操作 | 触发条件 |
|---|---|---|---|
| pending | 等待中 | 取消 | 创建时 |
| running | 执行中 | 取消（部分） | AI 开始处理 |
| completed | 已完成 | 查看结果、重新生成 | AI 返回结果 |
| failed | 失败 | 重试、查看错误原因 | AI 返回错误 |
| confirmed | 已确认（人工） | 使用/导出 | 人工确认 |

### 8.3 知识图谱节点（KnowledgeNode）

| 状态 | 说明 | 可执行操作 | 触发条件 |
|---|---|---|---|
| ai-generated | AI 生成 | 提交评审 | AI 生成时 |
| under-review | 评审中 | 评审专家审核 | 提交评审时 |
| published | 已发布 | 编辑（需重新评审） | 评审通过 |
| rejected | 已驳回 | 修订后重新提交 | 评审驳回 |

## 9. 交互动作矩阵

| 动作 | 入口 | 前置条件 | 确认 | 状态变化 | 数据影响 | 通知 | 失败回滚 |
|---|---|---|---|---|---|---|---|---|
| 上传资源 | GlobalDock /资源页 | 登录 | 文件格式校验 | 资源状态=上传中 | 写入 PostgreSQL resource 表 | 成功后提示 | 格式错误提示 |
| 发起裂变任务 | AISidebar | 资源已解析 | AI 确认 | AIMission=pending→running | 新增 mission 记录 | 完成后通知 | mission 状态=failed |
| 提交专业方案 | 专业空间 | 方案已完成编辑 | 表单校验 | 专业状态=drafting→submitted | 更新 professional_project 表 | 通知教务处 | 状态回退 |
| 构建知识图谱 | KnowledgeGraphView | 专业已发布 | 确认覆盖 | 节点状态=ai-generated | 写入 FalkorDB | 通知专业带头人 | 节点不保存 |
| 审批专业方案 | 管理空间 | 有待审批方案 | 审批意见填写 | submitted→first-reviewed 或驳回 | 更新状态+审批记录 | 通知申请人 | 无 |

## 10. 生命周期与流转规则

### 10.1培养方案（ProgramPlan）

- **创建**：专业带头人发起，关联专业
- **编辑**：起草阶段可自由编辑，提交后锁定
- **提交**：触发初审流程，状态变为 submitted
- **初审**：教务处管理员审核，通过则进入终审
- **终审**：评审专家审核，通过则发布
- **退回**：任何审核阶段可退回，状态回退并通知
- **归档**：年度结束后自动归档，历史数据只读

### 10.2 课程（Course）

- **创建**：专业带头人设计课程体系时创建
- **关联知识图谱**：课程节点自动关联图谱
- **发布**：关联专业审核通过后自动发布
- **作废**：需专业带头人申请，教务处管理员审批

## 11. 业务规则

| 规则 ID | 规则内容 |
|---|---|
| BR-001 | 资源上传大小上限：单文件 100MB |
| BR-002 | 支持格式：PDF、DOC/DOCX、PPT/PPTX、JPG/PNG |
| BR-003 | 长文裂变单次最大输入：50,000 字（超出自动分段） |
| BR-004 | AI 摘要最长输出：5,000 字 |
| BR-005 | 知识图谱节点评审时限：7 个工作日，超时自动催办 |
| BR-006 | AI 生成结果默认需人工确认，置信度 ≥ 0.95 可自动通过 |
| BR-007 | 同一用户不能同时属于教师空间和专业空间 |
| BR-008 | 培养方案编辑锁定：提交审核后至审核结束前不允许编辑 |

## 12. 假设

- SDS 系统数据将通过数据迁移工具导入，迁移粒度为"专业/课程/用户"
- GraphITI 引擎提供 REST API封装，Java 服务通过 HTTP 调用
- AI 大模型采用私有化部署，通过内部网关调用
- PgVector 与 PostgreSQL 共用同一数据库实例
- FalkorDB 与 Neo4j 为二选一，优先使用 FalkorDB（待技术选型确认）
