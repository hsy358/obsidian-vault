description: |
  手绘白板风格信息图，纵向泳道流程图（6 泳道 × 20 步）：
  - 用户 (User) / 蓝色
  - 智能体 (Agent) / 黄色
  - Agent Skills / 浅褐色
  - 大模型 (LLM) / 黄色
  - MCP (Model Context Protocol) / 粉色
  - 工具 (Tools) / 绿色

  20 步动作流（"之"字形往复循环，从左上用户输入开始 → 多泳道交互 → 左下返回用户）：
  ① 用户提问 → ② Agent 接收请求 → ③ Agent 历史对话/记忆检索 → ④ LLM 意图识别与任务拆解
  → ⑤ LLM 生成执行计划 → ⑥ Agent 接收计划 → ⑦ Agent 判断是否调用 Skills/工具（分叉）：
      分叉A：→ ⑧ 路由到匹配 Skill → ⑨ Skill 读取上下文与能力配置 → ⑩ Skill 组织调用策略与参数草案
      → ⑪ LLM 补全具体调用参数 → ⑫ Agent 通过 Skill 发起调用请求
      分叉B：→ ⑬ MCP 工具发现与选择（直接路径）
  → ⑭ MCP 调用具体工具 API → ⑮ 工具执行任务 → ⑯ MCP 接收结果 → ⑰ Skill 解析/校验/格式化结果
  → ⑱ Agent 整合 Skill 结果与上下文 → ⑲ LLM 生成最终回答 → ⑳ Agent 向用户返回结果

  关键设计点：
  - Skills 调用路径比 MCP 直连多了 3 步（路由 + 上下文配置 + 参数草案）—— Skills 是"加了策略层的 MCP"
  - 步骤 ⑦ 双分叉体现"是否需要领域专业知识"的决策逻辑
  - 步骤 ⑱ 后才回到 LLM 生成 = "工具调用结果融入上下文，再做 LLM 综合"
