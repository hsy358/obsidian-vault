Orca 的创始团队 Stably 把这套野路子工作流总结得很准：**2026 年多数「10x 开发者」都在同时跑 3–5 个编码 Agent**，但工具还停留在 tmux 分屏 + 手动 `git worktree add` 的阶段。Orca 要做的，就是把这个「你脑子里一直想搭却没搭好的编排壳」做出来。

## 二、核心定位：Agent 之上的「编排层」

这是 Orca 最容易被误解、也是最重要的设计哲学：

> **"Orca is the ADE for working with a fleet of parallel agents."**（Orca 是服务于「并行 Agent 舰队」的 AI 开发环境。）

理解这句话，要抓住三个关键词：

- • **ADE 不是 IDE**：传统 IDE（VS Code、Cursor）是「一个编辑器 + 一个 Agent」；Orca 是「一个编排台 + 任意多个 Agent」。它不写代码、不跑模型，它管的是**调度、隔离、对比、合并**。
- • **Agent 中立（agent-agnostic）**：Orca 明确表态「不卖某个 Agent，只卖它们之上的那一层」。官方支持 30+ 个 CLI Agent——Claude Code、Codex、Cursor、GitHub Copilot、Cline、OpenCode、Grok、Pi、Devin、Goose……原则是「只要能在终端里跑，就能进 Orca」。
- • **自带订阅（BYO Subscription）**：Orca 完全免费、开源（MIT），你只需要把自己已有的 Agent 订阅或 API Key 接进去。**它不抽成、不锁额度**，赚的是「编排」这个增量价值。

## 三、核心功能详解

### 1. 并行 Worktree：一个提示扇出 N 个 Agent

这是 Orca 的「杀手锏」。你可以把**同一个 prompt 同时发给 5 个 Agent**，每个 Agent 在各自**隔离的 git worktree** 里独立工作，互不污染工作目录；跑完之后你把 5 份实现摆在一起对比，挑最满意的合并。

### 2. 手机遥控：Agent 的「对讲机」

Orca 是少数把**移动端**当一等公民做的 Agent 工具。iOS（App Store / TestFlight）和 Android（APK） companion app 可以：

- • 实时**监控**每个 Agent 的状态，不挑终端；
- • Agent **完成时推送通知**，你不用死守电脑；
- • 在地铁上、被窝里**远程续发指令**，把「半夜想看进度」变成「手机点一下」。
这精准命中了长任务 Agent 的最大痛点：你派了一堆活，但人不能 24 小时钉在椅子上。

### 3. 设计模式 + 差异批注：让「视觉改动」闭环

这两件事把「改界面 → 看效果 → 提意见」的反馈环，整个收拢进同一个应用，不用再在浏览器、编辑器、聊天框之间反复横跳。

### 4. 原生 GitHub / Linear + SSH Worktree

- • **GitHub & Linear 原生集成**：在应用内浏览 PR、Issue、项目看板，从任意任务一键开 worktree 审查，不再切上下文。
- • **SSH Worktree**：把 Agent 跑在远端大算力机上，完整文件编辑 / git / 终端 / 端口转发 + 自动重连，本地只管看板和手机。

## 实际应用场景

- • **独立 10x 开发者**：一个需求扇出 5 个 Agent，喝杯咖啡回来挑最优实现合并，效率直接翻几倍。
- • **技术负责人**：手机收到「Agent 完成」推送，地铁上点开批注 diff，不在电脑前也能推进 review。
- • **远程 / 重负载团队**：SSH worktree 把 Agent 丢到机房大算力机，本地轻客户端 + 手机看板即可。
- • **Agent 横评**：同一任务同时派给 Claude Code / Codex / Cline 同台竞技，用真实 diff 说话，选型不再靠玄学。
- • **多账号玩家**：Account Switcher 直接看 Claude / Codex 的用量与限流重置时间，热切换账号无需重新登录。

## 快速上手

### 安装

```
# macOS（Homebrew）brew install --cask stablyai/orca/orca# Arch Linux（AUR）yay -S stably-orca-bin# 其他平台：去 onOrca.dev 下载 DMG / .exe / AppImage# 无界面服务器可跑：orca serve（headless Linux server）
```

### 第一条命令

```
# 在 Orca 内创建一个并行 worktreeorca worktree create feature/login# 给当前 worktree 打快照，方便回滚对比orca snapshot# 或在 GUI 里把同一个 prompt 直接扇出给 5 个 Agent
```

Orca 还暴露了 **Orca CLI**，让 Agent 也能反过来驱动 Orca 自己：`orca worktree create` / `snapshot` / `click` / `fill`——真正把「编排」也变成了可编程的接口。

> 💡 提示：Orca 本身免费，你只需接入已有的 Agent 订阅（Claude / Codex 等 API Key 或账号）。零Agent额度也能装，只是跑不起来活。
