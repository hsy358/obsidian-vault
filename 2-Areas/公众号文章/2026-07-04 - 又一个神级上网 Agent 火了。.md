## 有人会好奇，它是怎么做到的

**01 每一个平台都是一线和备用的多个后端路由。**

Agent Reach 由“工具集合”发展到“能力层”。

每个平台都有首选和备用接入方式，当前会自动帮你选最稳定的一套完成安装，并做完整可用性检测。

小红书主要使用OpenCLI桌面端，它直接复用了浏览器的登录态，几乎不需要做任何配置。

服务器环境用 xiaohongshu-mcp，通过扫码登录。实在不行再用 xhs-cli 做备选。

B站主要使用bili-cli，可以做到搜索、热门和详情，也不需要登录。

字幕部分就交给OpenCLI来处理了。最后使用搜索API作为兜底方案。

X 平台主要使用 x-cli，经过测试发现比较稳定。OpenCLI 作为浏览器登录态的备用方案。

![](https://mmbiz.qpic.cn/mmbiz_png/VDCUoW3UiblIMqKCnEmqviad5HkhZId8jqTHgM1k6M9j9Cxpx2gwiaMMR9rjiblH13TFYgbzEtmcLqgic7dDksD0IRvODlqF5ZicBvicYvJA4U0bJg/640?wx_fmt=png&from=appmsg)
整个接入方式换代,你无感。

**02 真体检,真探测。**

agent-reach doctor 真跑命令探测，并不是只看文件是否存在。

能够识别出“装了但是坏掉”的情况，例如Python升级后venv失效，并且给出了可以直接复制粘贴的重新安装方法。

并且还会输出各个后端实际的路径，还有`doctor --json` 中的`active_backend` 状态，方便找到当前所用的后端。

一个渠道出问题不会把整个体检拖垮，照样能跑完所有检查。

**03 13 个平台全覆盖。**

支持网页端、客户端、油管、Github、B站、小红书、RSS、V站、雪球等。

![](https://mmbiz.qpic.cn/sz_mmbiz_png/VDCUoW3UiblJZTZOU4Xy6jDaKVuUUAZEibtia4TQpOIqNvWX4uPzZlBf5e3icKp9SywMNGs8okU6DZbusclS86VIR6DcubH1PZuFgiaWzbiaqNuL4/640?wx_fmt=png&from=appmsg)
6 个真正的零配置：网页、油管、RSS、GitHub、V站、全网搜索，安装之后就可以用。

V站通过公开的JSON API来获取热门帖子、节点帖子、帖子详情和回复以及用户信息，并且不需要做认证或使用代理。

另外有 7 个是需要登录态或者 Cookie 的，但是桌面端 OpenCLI 可以直接复用浏览器的登录态。

如果你已经在浏览器里刷过小红书、登录过 Reddit，就可以直接使用。

OpenCLI 只要点击 Chrome 商店上的“添加扩展”按钮，在浏览器里已经登录过的平台都会被解除限制。

很适合用在开发、内容调研和行业资讯采集等三个主要的应用场景。

**04 完全免费、无API费用。**

所有工具都是开源的，所有的API都是免费的，本地电脑不用花一分钱。

不用花钱买API，也不需要获得任何API Key，更不用花钱。

所有的代码都是公开的，可以随时查看。

Cookie和Token都只存储在本地，文件权限是600，只有当前用户可以读写，不会被上传或者泄露出去。

另外安全模式--safe不会自动修改系统，只列出需要的东西让你自己选择是否安装。

## 看完这些功能，相信各位已经迫不及待了

我给大家说两种安装方式：

第一种打开claude code直接说：

```
帮我安装 https://raw.githubusercontent.com/Panniantong/agent-reach/main/docs/install.md
```

第二种手动命令行：

```
pip install https://github.com/Panniantong/agent-reach/archive/main.zipagent-reach install -e auto
```

安装完毕之后运行`agent-reach doctor`来检查所有的渠道状态。

然后打开Claude Code，说一句“帮我做一次MCP协议的全网调研”。

就可以使用了。

## 到这里缺点也给大家提提

**移除了三个渠道。**

抖音、微博、微信公众号已经下架了，因为上游已经停止维护或者被反爬拦截得很厉害。

因为反爬封杀，所以所有的开源工具都失效了，已经移除了，等上游恢复之后再添加回来。

**Reddit 必须有登录状态。**

Reddit 的匿名接口已经全部被封了，桌面端用户使用 OpenCLI，服务器端使用 rdt-cli 和 Cookie。

**只读,不操作。**

所有的操作都只是读取，并不会发布帖子、点赞或者关注，有这个需求的朋友可以出门左拐。

## 写在最后

Agent Reach 不适合所有人。

如果只是偶尔使用 AI 聊天、写文案的话，那么就不需要去折腾它了。

但是如果你经常要让Agent帮你做调研、跨平台汇总信息、监控某个话题的动态的话，那么这个工具可以为你节省很多时间。

特别是开发者，内容创作者和做市场调研的人会认为很有用。

有需要的朋友可以装上试试。
