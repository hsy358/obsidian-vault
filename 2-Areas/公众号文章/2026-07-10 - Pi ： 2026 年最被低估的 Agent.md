# Pi ： 2026 年最被低估的 Agent

![封面](https://mmbiz.qpic.cn/sz_mmbiz_jpg/IfsstJhWGiaJjXSTcTDdvmEvXiahbkmbjL3RpWB1ZCNBoCkgdwK4opPuD5Fm0g9KtbVLMSDNFV89fXAJwqEwREibnHWJxN8SZVyXZuibze6oJtQ/0?wx_fmt=jpeg)

2026 年我觉得最被低估的 Agent 框架，是 Pi。

大部分人用 Claude Code，其实也就用到了它一半、甚至更少的功能。剩下的那一堆内置能力，你不用它也在那儿默默吃 token，上下文越来越脏，模型也越来越不专注。钱花了，劲没使对地方。

我这段时间一直把 Pi 当自己开发 Agent 的核心框架来用，体感完全不一样。
Pi 的脾气很合我——less is more。它默认就 4 个工具：Read、Write、Edit、Bash，系统提示词也极简。就这么 4 个，其实已经覆盖了 Agent 绝大部分的使用场景。

但 Pi 真正狠的地方不是少，是能长。

它还带着一套完整的 Harness SDK，叫 pi-agent-core。想在上面做二次开发、快速搭一个完全属于自己的智能体，特别顺手。有朋友跟我讲，这是他用过最好的 AI SDK，事件驱动的设计，搭复杂 UI 或者自定义流程的时候，手感很舒服。

Pi 最对味就一点：它不试图替你做所有事。它给你一个干净的底座，剩下的，你自己去把它拓成最适合你的那个工具。
