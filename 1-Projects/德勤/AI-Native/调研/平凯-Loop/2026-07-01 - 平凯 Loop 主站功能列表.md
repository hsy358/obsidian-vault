# 平凯 Loop 主站功能列表（2026-07-01）

> 评估：能否作为德勤 v0.3 的**第二执行器**（SaaS 形态）+ **R4 协作** 借鉴对象

## 一、产品定位

> **Agent and person — Collaborate like colleagues**
> AI teammates, in the chat your team already uses.

## 二、产品能力（PRODUCT CAPABILITIES）

| 功能 | 说明 | 德勤对应 |
|---|---|---|
| **Give Agents clear responsibilities in team delivery** | 按权限+上下文唤醒 Agent，团队看结论/风险/决策 | R3 执行器 + R4 协作 |
| **Unified management of agents in teams** | Agent 所有权、职责、可见性统一管 | Table 16 Row 3 Agent 管理 |
| **Task continues after person leaves computer** | 人离机后任务继续推进 | Table 16 Row 2 任务看板（执行中状态） |
| **Complex tasks broken down in parallel** | 任务拆解 + 并行推进 | 同上 |
| **Agent reach determined by permissions** | @ 范围由权限决定 | R7 治理 |
| **Report regularly like a team member** | 固定节奏同步进度+风险 | Table 17 Row 6 基础日志 |

## 三、效率提升数字（来自主站对比表）

| 维度 | 传统人工编程 | 传统 Vibe Coding | 平凯 Loop |
|---|---|---|---|
| 交付速度 | 1x | 5x-10x | **20x-50x** |
| 人的效率 | 1x | 1.7x-2.5x | **4x-10x** |
| 跟进效率 | 1x | 1.8x-2.9x | **6.7x-20x** |
| 协作沟通效率 | 1x | 1.3x-1.7x | **4x-10x** |

> **内部估算**（不保证科学严谨），但说明 Loop 把"多 Agent 协作 + 任务并行"作为核心叙事。

## 四、企业级控制边界（ENTERPRISE-READY）

| 控制点 | Loop 实现 | 德勤对应 |
|---|---|---|
| 普通消息默认**不**群发给 Agent | ✅ 只在 @ / 话题参与 / 任务分配时唤醒 | 类似 Table 17 "可观测与日志" |
| @all / @online 显式广播控制 | ✅ 仅在支持的频道 + 权限管控 + 二次确认 | R7 安全审批 |
| 个人私有上下文不外溢 | ✅ 需引用或主动交付 | R7 数据权限边界 |
| 交付诊断可追溯 | ✅ 消息保留投递快照：哪些 Agent 命中/排除/唤醒/原因 | R6 可观测 + R7 审计 |

## 五、并行能力

- 单线处理（手动排队）→ 本地加速 → **多任务多 Agent 同时处理**（Loop 强项）
- 上下文传递：分散 → 每轮手动复制 → **频道+话题+任务共享同一上下文**
- 偏差成本：手动维护 → 长对话漂移 → **任务板固定范围+负责人+验收**
- 交付记录：事后整理丢过程 → Prompt 结果难复用 → **话题/任务/执行/汇报持续记录**
- 安全边界：靠流程 → 手动粘贴边界不清 → **精确投递+显式广播+权限+诊断**

## 六、典型场景（GET STARTED 推荐）

1. **研发 walkthrough**
2. **客户项目**
3. **运营报告**

> **人负责方向+验收，Agent 负责执行+反馈，团队看结论/风险/下一步**

## 七、关键洞察（德勤 v0.3 评估）

### 🟢 Loop 真正强的 3 件事

1. **任务并行 + 多 Agent 协同执行**（"任务板 + 话题" 是真的，不是 PPT）
2. **企业级控制边界**（@范围/广播/私有/诊断四件套）
3. **交付记录可追溯**（不是事后整理丢过程）

### 🔴 Loop 缺失的 4 件事（v0.3 必做）

1. **审批节点**（关键交付物必须人确认）— Loop 没有
2. **私有部署**（Loop 是 SaaS，v0.3 要 R9 可单独部署）
3. **深度数据模型开放**（Loop 是封闭产品）
4. **可观测度量**（Loop 内部有，外部不可见；v0.3 要 Langfuse 级）

### 🟡 Loop 给德勤的"借鉴清单"（按优先级）

| 优先级 | 借鉴对象 | 用在哪 |
|---|---|---|
| 🔥 高 | **任务板 + 话题 = 协作最小单元**（Loop 的核心 UX）| R4 协作 / 项目首页 |
| 🔥 高 | **@ 范围权限管控 + 投递诊断** | R7 治理 |
| 🟢 中 | **执行器后端选型（Codex/Claude Code/OpenCode）** | R3（已与德勤选型一致）|
| 🟢 中 | **机器/智能体/运行实例 三层拆分** | R3 执行器抽象 |
| 🟡 低 | **效率提升数据展示**（20x-50x）| 沟通话术 + v0.3 商业论证 |

## 八、是否试用？

- **5 分钟注册 + 跑通 5 步** 即可
- 重点验证：
  1. 任务并行是否真的并行（多 Agent 同时跑一个任务）
  2. @权限管控是否真的能阻止误唤醒
  3. 投递诊断快照长什么样（v0.3 的 R6/R7 应该参考）
- 试用后 → 我可以更新 R3/R4 借鉴深度
