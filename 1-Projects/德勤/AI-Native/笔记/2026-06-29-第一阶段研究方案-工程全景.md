## 📋 R1. 数据模型 + Schema 设计

**研究目标**：客户/项目/任务/Agent/执行器/审批/审计 8 张核心表的 schema 设计

**候选参考**：
- **Paperclip**（⭐72k）—— 组织树 + 目标对齐 + 预算 + 心跳 + 治理 + 审计 + BYOA（最完整参考实现）
- **AgentSpace**（⭐547）—— 企业 Workspace + 数字员工 + 权限 + 审批 + AgentRouter
- 报告"必须自研 9 项"第 1 项：**统一产品数据模型**（数据模型必须自研，开源参考）

**产出**：
- `/tmp/paperclip/` 看代码，**借鉴组织树 / 心跳 / BYOA 设计**
- 写出"德勤版数据模型 schema v0.1"（v0.3 文档里的 8 张表 + ER 图）
- 决策：完全自研 vs fork Paperclip 改造

**资源**：⚠️ Paperclip 110 MB（仅看代码，不部署）  
**预计耗时**：3-4 天

## 🎨 R2. 看板 UI + 协作界面

**候选参考**：

**产出**：
- "3 类看板 UI 设计稿"（可用 React + Ant Design / shadcn）
- 与 Hermes kanban 数据库对接（write 同步）
- 可选：Web UI 嵌入 Obsidian vault 渲染（用 vault PARA 已有内容）

**资源**：⚠️ React 项目约 200MB（node_modules）+ 磁盘  
**预计耗时**：5-7 天（含 UI 设计 + 编码）

## 🔌 R3. 执行器抽象层（**关键！**）

**研究目标**：设计**统一执行器接口**，让 **Hermes / OpenClaw / Codex / Claude Code 等都接得上**

```
┌────────────────────────────────────────┐
│  Executor Abstract Interface (自研)    │
│  - submit(task)                        │
│  - heartbeat()                         │
│  - logs()                              │
│  - pause() / resume() / terminate()    │
└──────────────┬─────────────────────────┘
               │
   ┌───────────┼───────────┬──────────────┐
   │           │           │              │
   ▼           ▼           ▼              ▼
Hermes      OpenClaw    Codex       Claude Code
 chat        gateway    CLI        SDK
 adapter     adapter    adapter     adapter
```

**候选参考**（借鉴实现细节）：
- **LangGraph**（⭐36k）—— StateGraph + checkpoint + memory + tool calling 模式
- **Claude Code SDK** —— Anthropic 官方 Python SDK
- **OpenClaw** —— `sessions_spawn` API（已用过，熟悉）
- **Codex CLI** —— OpenAI 官方 CLI

**产出**：
- `/root/vault/1-Projects/德勤/AI-Native/executor/abstract-interface.md`（接口设计）
- `/root/vault/1-Projects/德勤/AI-Native/executor/adapters/hermes-adapter.py`（参考实现）
- `/root/vault/1-Projects/德勤/AI-Native/executor/adapters/claude-code-adapter.py`（参考实现）
- `/root/vault/1-Projects/德勤/AI-Native/executor/adapters/langgraph-adapter.py`（参考实现）

**资源**：✅ **完全够用**（pip install + Python 代码）  
**预计耗时**：5-7 天（4 类 adapter 各 1-2 天）

## 🐳 R9. 部署架构（模块化 + 容器化 + 单独部署）

**研究目标**：**整套系统可以单独部署**（每个组件独立部署 + 整体可打包）

**候选参考**：
- **Langfuse docker-compose** —— 多服务容器化部署模板
- **Paperclip 自托管指南** —— Next.js + Postgres 部署
- **Hermes v0.14 自带 gateway** —— 单一进程可跑

**产出**：
- `/root/vault/1-Projects/德勤/AI-Native/deployment/architecture.md`（架构图）
- `/root/vault/1-Projects/德勤/AI-Native/deployment/docker-compose.yml`（整套启动）
- 每个组件独立 Dockerfile（executor / vault / kanban / observability / memory）

**部署形态**：
```
┌────────────────────────────────────────────────┐
│  docker-compose up  →  整套系统启动              │
│                                                │
│  ┌──────────┐  ┌──────────┐  ┌──────────┐      │
│  │ Hermes   │  │ Executor │  │ Kanban   │      │
│  │ Gateway  │  │ Gateway  │  │ API      │      │
│  │ :18789   │  │ :8001    │  │ :8002    │      │
│  └──────────┘  └──────────┘  └──────────┘      │
│  ┌──────────┐  ┌──────────┐  ┌──────────┐      │
│  │ Memory   │  │ Observ.  │  │ Vault    │      │
│  │ Provider │  │ (SDK)    │  │ PARA     │      │
│  │ :8003    │  │ (no svc) │  │ :8004    │      │
│  └──────────┘  └──────────┘  └──────────┘      │
│  ┌──────────────────────────────────────┐      │
│  │ 统一前端（React + Ant Design）       │      │

└────────────────────────────────────────────────┘
```

**资源**：⚠️ 6 个 Docker 容器约需 4-6 GB 内存（**本服务器 2.4 GB 不够完整跑**，需分阶段）  
**预计耗时**：2-3 天（架构图 + docker-compose 模板）
