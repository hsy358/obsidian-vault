### 德勤执行器 adapter 借鉴（伪代码）

```python
# /root/vault/1-Projects/德勤/AI-Native/executor/langgraph-adapter.py
from langgraph.graph import StateGraph, START, END
from langgraph.checkpoint.memory import MemorySaver

class DeloitteAgentState(TypedDict):
    task_id: str
    project_id: str
    messages: list
    step_count: int
    context: dict
    artifacts: list
    approvals_needed: bool

def research_node(state): ...  # 调 LLM API
def analyze_node(state): ...
def approval_node(state): ...  # 人工审批
def deliver_node(state): ...  # 生成交付物

graph = StateGraph(DeloitteAgentState)
graph.add_node("research", research_node)
graph.add_node("analyze", analyze_node)
graph.add_node("approval", approval_node)
graph.add_node("deliver", deliver_node)
# ... add_edge / add_conditional_edges
app = graph.compile(checkpointer=MemorySaver())
```

### Adapter 接口规范（AdapterExecutionContext + Result）

```ts
interface AdapterExecutionContext {
  runId: string;
  agent: { id: string; companyId: string; name: string; adapterConfig: unknown };
  runtime: { sessionId: string | null; sessionParams: ... };
  config: Record<string, unknown>;
  context: Record<string, unknown>;
  onLog: (stream: "stdout" | "stderr", chunk: string) => Promise<void>;
  onMeta?: (meta: AdapterInvocationMeta) => Promise<void>;
  onSpawn?: (meta: { pid: number; startedAt: string }) => Promise<void>;
}

interface AdapterExecutionResult {
  exitCode: number | null;
  signal: string | null;
  timedOut: boolean;
  errorMessage?: string | null;
  usage?: { inputTokens: number; outputTokens: number };
  sessionParams?: Record<string, unknown> | null;
  sessionDisplayId?: string | null;
}
```

### 架构（AGENTS.md）

```
server/                    Express REST API + 编排服务
ui/                        React + Vite 看板 UI
packages/db/               Drizzle schema + 迁移
packages/shared/           共享类型/常量
packages/adapters/         Agent adapter 实现 ← 借鉴点
packages/adapter-utils/    共享 adapter 工具
packages/plugins/          插件系统
doc/                       运维/产品文档
```

### 关键 Invariant（adapter 必须遵守）

```
1. **No-remote-git contract**：adapter 不能 git push 到 remote
2. **Local execution workspace** 是唯一跨 run 持久化边界
3. 每次 run 给出 local cwd，agent 在 cwd 里工作
4. run 结束时改回 cwd
5. 失败的 sync-back 是 run-level error（gate dependent wakes）
```

## 🎯 4 个研究包综合结论

### 关键发现

1. **Hermes 框架在所有竞品中都被支持**：Paperclip / AgentSpace 都已实现 Hermes adapter，**何大人的"选 Hermes"决策被市场验证**
2. **多 Agent 框架并存是行业共识**：Paperclip 13 adapter + AgentSpace AgentRouter 5 runtime —— 不是"二选一"
3. **数据模型都是 Drizzle**：Paperclip / AgentSpace 都用 Drizzle ORM，**德勤 R1 也应该用 Drizzle（不是 SQLAlchemy）**
4. **本地代码 = git worktree**：Paperclip 强制"no-remote-git contract"——adapter 不能 push，**这是德勤 sandbox 应该学的安全模型**
5. **Langfuse 在本机不现实**：Self-hosted 重 + 2.4 GB RAM 限制，**MVP 阶段必须用 SDK + 后期考虑 Cloud**

### 服务器资源决定

| 开源 | 本机装？ | 替代方案 |
|---|---|---|
| LangGraph | ✅ 装 + demo 跑通 | 直接用 |
| Paperclip | ❌ 不装（需要 Postgres + Next.js）| 只读代码（已完成）|
| AgentSpace | ❌ 不装（需要 Postgres + Next.js）| 只读代码（已完成）|
| Langfuse | ⚠️ SDK 装，server 不装 | Langfuse Cloud / Phoenix SDK |

