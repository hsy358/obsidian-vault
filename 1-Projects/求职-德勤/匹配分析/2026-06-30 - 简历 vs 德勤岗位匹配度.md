## 一、德勤岗位核心画像（从开发说明提取）

### 产品定位
> **不是单点 Agent 工具，是组织级 AI 工作空间**——把人、Agent、任务、Skill、MCP、上下文、执行过程、审批与交付物放到同一个 Workspace 中可视化、可管理、可沉淀。

### 10 大核心模块
1. **项目空间**（Workspace）— 组织/项目/成员/资料
2. **目标与任务看板** — 任务拆解 + 状态流转
3. **Agent 管理** — 数字员工 + 角色 + Skill + 执行器
4. **Skill / MCP 管理** — 资产定义 + 运行使用
5. **协作沟通** — 频道 + 线程 + @Agent
6. **运行监控看板** — Agent 状态 + 日志 + 调用链
7. **交付物管理** — PPT/Word/Excel + 版本 + 审批
8. **审批与风险** — 关键结论 + 敏感操作
9. **Memory / Context** — 项目背景 + 沉淀候选
10. **运营管理看板** — 进度 + 质量 + 风险

### 5 大典型场景
- 企业 AI Native 转型咨询（资料收集 → 现状分析 → 方案设计 → PPT/Word 初稿）
- Physical AI 项目前期评估
- 行业研究与方案复用
- 交付复盘与知识沉淀

### 技术栈需求
| 层 | 技术 |
|---|---|
| **执行器** | Hermes / OpenClaw / Claude Code / Codex / NemoClaw / LangGraph |
| **工作流** | LangGraph（StateGraph）|
| **可观测** | Langfuse / OpenTelemetry |
| **Memory** | Mem0 / honcho |
| **RAG** | RAGFlow / Dify |
| **协作** | OpenAgents 频道 + 线程 |
| **数据模型** | Drizzle ORM |
| **看板** | Vibe Kanban（参考）|
| **沙箱** | NemoClaw / OpenShell |

### MVP 11 任务（1-2 月）
- **M1** 项目 Workspace 骨架（数据模型 + 成员 + Agent 档案）
- **M2** 任务看板 6 状态流转（待拆解/待执行/执行中/待确认/已完成/失败阻塞）
- **M3** Agent 管理（profile + 角色 + Skill + 执行器）
- **M4** Skill/MCP 基础登记
- **M5** 基础协作（频道 + 线程 + 文件共享）
- **M6** 执行与日志（2 类执行器）✅ 已跑通 LangGraph adapter
- **M7** 交付物管理（PPT/Word/Excel）
- **M8** 人工审批
- **M9** 项目 Memory
- **M10** Demo 场景（1-2 月）
- **M11** 项目首页 + 抽屉（React）

### 阶段路线
| 阶段 | 时间 | 重点 |
|---|---|---|
| 1 | **1-2 月** | MVP 跑通（11 任务端到端）|
| 2 | 2-4 月 | 看板产品化（运行/协作/调用链/审批/交付物看板）|
| 3 | 4-8 月 | 知识/Memory/安全治理 |
| 4 | 8-12 月 | Physical AI 集成 |

5. **主动提"开源生态熟悉度"**：你在 vault 里已经在用 Hermes + OpenClaw + LangGraph + Paperclip + AgentSpace，这是稀缺经验，**主动提**。
