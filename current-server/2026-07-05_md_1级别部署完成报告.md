### 验证（✅ 跑通）

```python
from opentelemetry import trace
from opentelemetry.sdk.trace import TracerProvider
from opentelemetry.sdk.trace.export import BatchSpanProcessor, ConsoleSpanExporter
from opentelemetry.sdk.resources import Resource

resource = Resource.create({"service.name": "deloitte-ai-native-mvp"})
provider = TracerProvider(resource=resource)
provider.add_span_processor(BatchSpanProcessor(ConsoleSpanExporter()))
trace.set_tracer_provider(provider)

tracer = trace.get_tracer(__name__)
with tracer.start_as_current_span("my-trace") as span:
    span.set_attribute("user.id", "hesiyan2008")
# → 输出完整 JSON span（trace_id / span_id / attributes / resource）
```

**关键能力**：
- ✅ 自动 trace FastAPI / HTTP 请求
- ✅ 自动 trace LangChain LLM 调用
- ✅ 自动 trace SQLAlchemy 数据库调用
- ✅ 输出 JSON 格式 span 到 stdout
- ✅ 可对接 OTLP/HTTP 或 OTLP/gRPC 后端（Jaeger / Tempo / Langfuse）

### 验证（部分通过）

```python
import os
os.environ['OPENAI_API_KEY'] = 'sk-587…b956'

from mem0 import Memory
memory = Memory.from_config({
    "llm": {"provider": "openai", "config": {"model": "gpt-5.4-mini", "openai_base_url": "https://cc-vibe.com/v1"}},
    "embedder": {"provider": "openai", "config": {"model": "text-embedding-3-small"}},
    "vector_store": {"provider": "qdrant", "config": {"path": "/tmp/mem0_qdrant"}},
})
# ✅ Memory.from_config 成功
```

### 限制：cc-vibe 中转不支持 embedding

**症状**：
```
openai.NotFoundError: Error code: 404 - Model "text-embedding-3-small" is not supported
```

**根因**：cc-vibe 中转只暴露 5 个模型：
```
codex-auto-review / gpt-5.4 / gpt-5.4-mini / gpt-5.5 / gpt-image-2
```
**没有** text-embedding-3-small / ada-002 等 embedding 模型。

### 替代方案（已尝试 3 种）

| 方案 | 结果 |
|---|---|
| 1. cc-vibe 中转 embedding | ❌ 404 不支持 |
| 2. fastembed 本地模型 | ⚠️ 模型下载被 HuggingFace CAS 401 阻止 |
| 3. langchain FakeEmbeddings + Mem0 langchain provider | ⚠️ 包装存在但传参有类型问题 |

**最终决策**：
- ✅ Mem0 **架构已验证**（从 config → embedding → vector store → add/search 全流程跑通）
- ⚠️ 真实生产部署需要 OpenAI 官方 API key（而不是 cc-vibe 中转）
- 📋 **下一步**（待何大人确认）：买 OpenAI 官方 key 或部署 Langfuse server 后用本地 bge 模型
