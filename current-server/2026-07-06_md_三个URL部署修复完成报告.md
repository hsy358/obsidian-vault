## 🔧 1. Langfuse 修复（5 个配置错误）

### 修复清单

| # | 问题 | 修复 |
|---|---|---|
| 1 | ports 绑 `127.0.0.1:3000`（公网不可达）| 改 `0.0.0.0:3000:3000` |
| 2 | `host.docker.internal:5432` Linux Docker 解析失败 | 加 `extra_hosts: ["host.docker.internal:host-gateway"]`（Docker 29 语法） |
| 3 | `ENCRYPTION_KEY=000…000` 被 v3 拒绝（必须 256 bit 随机） | 改 `openssl rand -hex 32` 生成 |
| 4 | worker 缺 `LANGFUSE_S3_EVENT_UPLOAD_BUCKET` | 加 `langfuse` |
| 5 | **ClickHouse 用 ReplicatedMergeTree 需要 Zookeeper 集群** | 加 `CLICKHOUSE_CLUSTER_ENABLED=false`（切到 unclustered 迁移 + MergeTree） |

### 顺手修的基础设施

| # | 问题 | 修复 |
|---|---|---|
| A | `postgres` 超级用户无密码（pg_hba scram-sha-256 拒绝） | `ALTER USER postgres WITH PASSWORD 'postgres'` |
| B | PG `listen_addresses = 'localhost'`（容器内连不上） | 改 `'*'` |
| C | pg_hba.conf 没加 docker 网段（172.17-172.31）| 加 `host all all 172.x.0.0/16 scram-sha-256`（15 行） |
| D | Redis `bind 127.0.0.1`（容器内连不上） | 改 `bind 0.0.0.0 -::1` |

### compose 改后关键片段

```yaml
services:
  langfuse-web:
    extra_hosts:
      - "host.docker.internal:host-gateway"
    ports:
      - "0.0.0.0:3000:3000"
    environment:
      DATABASE_URL: postgresql://postgres:postgres@host.docker.internal:5432/langfuse
      ENCRYPTION_KEY: d434f3ff62218c025ae5e4e825312715c4f7facca68d4751b4e2aecf9a97d297  # 真随机
      LANGFUSE_S3_EVENT_UPLOAD_BUCKET: langfuse
      CLICKHOUSE_CLUSTER_ENABLED: "false"  # 单机模式关键
```

## 📚 经验沉淀（避免未来重蹈覆辙）

### 教训 1：Docker + 宿主机共享 DB 的标准坑

```
❌ 默认做法：docker compose 起内置 postgres + redis（端口冲突 + 资源浪费）
✅ 正确做法：用现有 PG/Redis + extra_hosts: host-gateway
   - PG: listen_addresses='*' + pg_hba 加 docker 网段
   - Redis: bind 0.0.0.0
   - compose: extra_hosts: ["host.docker.internal:host-gateway"]
```

### 教训 2：ClickHouse 集群模式默认需要 Zookeeper

```
❌ 启单节点 ClickHouse + 默认 Langfuse → 失败 (ReplicatedMergeTree)
✅ 加 CLICKHOUSE_CLUSTER_ENABLED=false → 用 MergeTree（unclustered 迁移）
```

### 教训 3：端口冲突的"小问题大排查"

```
❌ 只看端口是否监听 → 漏掉 host port vs container port
✅ 改 .env EXPOSE_*_PORT=空 → docker compose 自动随机分配
   （服务间用容器名连，不依赖 host port）
```

### 教训 4：compose profiles 是必学工具

```
❌ 默认 docker compose up -d 启一堆内置服务（重复）
✅ docker compose up -d <service1> <service2> ... 显式指定
   或加 profiles: ["disabled"] 让默认跳过
```
