# 产品需求分析 / Product Analysis

**文档：** Yuanzhi OS 竞品架构映射与 AI 重构解析
**版本：** v1.0
**日期：** 2026-06-09
**Gate 状态：** ⚠️ BLOCKED（当前用默认假设推进）

---

## 1. 业务背景与问题

### 1.1 当前痛点

原 SDS 教学管理系统存在以下核心问题：

1. **表单驱动而非场景驱动**：用户在处理连贯业务时需在不同菜单间反复跳转，高度依赖人工逐行填报
2. **AI 能力缺失**：所有数据靠人工录入，无自动抽取、无智能辅助、无知识沉淀
3. **菜单冗余**：竞品将同一业务拆成多个二级菜单（如项目管理拆成 6 个同级菜单）
4. **知识断链**：年度目标填完即闲置，无法追踪落地；课程与毕业要求之间缺乏关联

### 1.2 解决方案愿景

Yuanzhi OS 的核心愿景：**从表单驱动转向场景闭环与 AI 原生驱动**

- 按业务闭环重组功能为三大空间（管理空间、专业空间、教师空间）
- 将 AI 引擎（基于大模型与 GraphITI 知识图谱）深度嵌入业务流
- 实现"系统自动抽取、AI 辅助生成、图谱自动沉淀"

## 2. 业务目标与成功指标

### 2.1 核心业务目标

| 目标 | 描述 | 优先级 |
|---|---|---|
| **全面替代 SDS 系统** | 在 L1~L3 级别覆盖 SDS 所有功能，并通过 AI 能力实现体验升级 | P0 |
| **降低人工填报负担** | AI 自动抽取 PDF 指标、OCR 自动打标，减少人工录入 | P0 |
| **实现知识图谱沉淀** | 通过 GraphITI 将课程知识自动生成知识网 | P1 |
| **提升任务追踪效率** | AI 自动裂变长文为可追踪微任务卡片 | P1 |

### 2.2 成功指标（待量化）

> ⚠️ **待确认：** 以下指标为建议值，需业务方确认量化目标。

| 指标 | 描述 | 建议目标 |
|---|---|---|
| 功能覆盖率 | Yuanzhi OS L1~L3 功能覆盖 SDS 比例 | ≥ 95% |
| AI 指标提取成功率 | PDF 上传后成功提取三级指标树的比例 | ≥ 90% |
| 任务裂变准确率 | AI 裂变任务卡片与人工判断一致的比例 | ≥ 85% |
| OCR 字段识别准确率 | 多模态 OCR 抽取字段与人工核对一致的比例 | ≥ 90% |
| 用户满意度 | 三大空间用户满意度调研 | ≥ 4.0/5.0 |

### 2.3 非目标（Out of Scope）

以下内容明确不在首版范围内：

- 移动端适配
- 多语言/国际化
- 积分兑换奖励系统（首版仅展示积分）
- 跨校数据互通
- 离线客户端
- 完整 BPMN 流程引擎（首版仅单节点审批）

## 5. 主流程、分支流程与异常流程

### 5.1 管理空间主流程

**US-M01：全局数据大盘**
1. 校领导登录 → 进入全局数据大盘（核心视图）
2. 系统展示全校概览指标（学生数/教师数/开课率等）
3. 校领导通过自然语言向 Data Copilot 提问
4. Data Copilot 调用预置 SQL 模板（ASSUM-001）生成数据透视图表
5. 校领导查看图表，导出数据报告

> ⚠️ **澄清中（ASSUM-001）：** Text-to-SQL 首版仅支持预置 SQL 模板。

**US-M02：评价考核中枢**
1. 教务处管理员上传校级教学标准 PDF
2. AI 引擎解析 PDF，提取三级指标树
3. 系统展示指标树（草稿状态）
4. 管理员人工确认指标树内容
5. 确认后指标树转为正式状态，纳入考核评价

> ⚠️ **澄清中（ASSUM-005）：** AI 提取失败时提示手动导入，不自动卡死流程。

**US-M03：底层配置底座**
1. 系统管理员进入表单设计器
2. 管理员通过拖拽调整字段顺序、设置必填/非必填（ASSUM-003）
3. 管理员设计审批流程（首版仅单节点审批）
4. 保存并发布表单/审批配置
5. 普通用户在业务流中填写表单或发起审批

### 5.2 专业空间主流程

**US-P01：人培方案研制**
1. 系部主任/专业带头人进入人培方案编辑器
2. 创建/编辑 OBE 矩阵（毕业要求 ↔ 课程体系支撑度）
3. AI 辅助检查合规性（CoPo 支撑度打分）
4. 提交评审
5. 评审专家提供评审意见（ASSUM-004）
6. 发布人培方案

**US-P02：建设规划长文裂变**
1. 系部主任上传年度建设规划文档（长文）
2. 点击"AI 分解"按钮
3. AI 按语义段落裂变为微任务卡片（ASSUM-002）
4. 管理员/系部主任为卡片分配负责人和截止日期
5. 任务负责人领取任务，执行并更新状态
6. 任务完成后由系部主任验收

**US-P03：教科研项目流水线**
1. 创建项目（填写项目信息、预算）
2. 项目立项后进入 Kanban 看板
3. 团队成员在卡片上更新进度
4. 上传中期汇报 PDF
5. 点击"AI 提炼里程碑"，AI 提取里程碑（ASSUM-010）
6. 用户确认后 AI 自动扣减预算
7. 项目结项，结项后归档

### 5.3 教师空间主流程

**US-T01：四宫格工作台**
1. 教师登录 → 进入四宫格工作台
2. 查看个人待办（来自各空间的任务）
3. 收到审核单据，唤起侧边栏 AI Agent
4. AI Agent 辅助撰写审核意见初稿
5. 教师确认/修改后一键提交

**US-T02：课程标准研制与知识图谱**
1. 教师进入课程空间，选择课程
2. 输入核心知识点
3. AI 拉取人培方案矩阵，生成课程标准（草稿态）（ASSUM-007）
4. 教师人工确认课标内容，发布课标
5. 手动触发 GraphITI 生成课程知识图谱（ASSUM-008）
6. 图谱发布，供其他教师和学生查阅

**US-T03：教师档案袋**
1. 教师进入档案袋，上传 PDF/图片证明文件
2. 多模态 AI（Qwen-VL/ GPT-4o）OCR 抽取字段（ASSUM-009）
3. 抽取结果展示给教师确认
4. 确认后成果计入积分账户（ASSUM-006）
5. 档案袋按类型归档

### 5.4 异常流程（通用）

| 场景 | 处理方式 |
|---|---|
| AI 任务裂变失败 | 提示用户"AI 分解失败，请手动创建任务卡片" |
| PDF 解析超时（>30s） | 提示用户"文档较大，请在稍后查看结果"；异步处理 |
| OCR 识别置信度低（<0.7） | 标红低置信度字段，提示人工核对 |
| 表单提交时字段校验失败 | 前端实时校验，阻止提交并高亮错误字段 |
| 重复提交 | 后端幂等校验，提示"该表单已提交，请勿重复操作" |
| 任务负责人失联（超期未完成） | 状态保留，通知系部主任重新分配 |
| 预算扣减超支 | 提示警告，阻止超支操作，需上级审批 |

## 6. 业务规则

### 6.1 权限与数据范围规则

| 规则 | 说明 | 证据等级 |
|---|---|---|
| 校领导数据范围 | 可查看全校数据，不可编辑 | assumed |
| 教务处管理员数据范围 | 可管理全校数据，编辑管理空间配置 | assumed |
| 系部主任数据范围 | 仅可操作本系/本专业数据 | assumed |
| 一线教师数据范围 | 仅可操作本人数据 | assumed |
| 评审专家数据范围 | 专业空间只读+提交评审意见 | assumed |
| 系统管理员数据范围 | 仅管理空间配置，无业务数据权限 | assumed |

### 6.2 AI 行为规则

| 规则 | 说明 | 证据等级 |
|---|---|---|
| AI 指标提取兜底 | AI 失败时提示手动导入，不自动卡死 | assumed |
| AI 课标生成状态 | 生成内容存草稿态，人工确认后转正式 | assumed |
| AI 里程碑触发 | 用户手动触发，用户确认后扣减预算 | assumed |
| AI 任务裂变粒度 | 按语义段落裂变，每段一张卡片 | assumed |
| GraphITI 图谱范围 | 课程级手动触发生成 | assumed |

### 6.3 积分规则（首版）

| 规则 | 说明 | 证据等级 |
|---|---|---|
| 论文积分 | 按发表期刊级别乘以系数（核心/普刊不同） | assumed |
| 专利积分 | 按专利类型（发明/实用新型/外观设计）乘以系数 | assumed |
| 教改积分 | 按项目级别（国家级/省级/校级）乘以系数 | assumed |
| 积分用途 | 首版仅展示累计，不做兑换 | assumed |
