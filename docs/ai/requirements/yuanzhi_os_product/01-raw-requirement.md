# Yuanzhi OS 竞品架构映射与 AI 重构解析

本模块定义了 Yuanzhi OS 如何在 L1~L3 级别全面覆盖传统竞品 (SDS 教学管理系统) 的功能，并通过 AI 工作流、知识图谱与空间架构实现体验的"降维打击"。

## 第一章 竞品架构极细粒度映射与 AI 重构解析

### 1.1 重构理念与 AI 赋能解析

原 SDS 系统采用了传统的"表单驱动 + 菜单堆砌"架构，用户在处理连贯业务时需在不同菜单间反复跳转，且高度依赖人工逐行填报。

本次重构的核心在于**"从表单驱动转向场景闭环与 AI 原生驱动"**。我们将所有功能打散，按业务闭环重新聚合成**管理空间**、**专业空间**和**教师空间**。同时，将 AI 引擎（基于大模型与 GraphITI 知识图谱）深度嵌入业务流，实现从"人工填表"到"系统自动抽取、AI 辅助生成、图谱自动沉淀"的跨越。

### 1.2 核心模块极细粒度映射表 (L1~L3)

#### 📌 空间一：🏢 管理空间

| Yuanzhi OS 架构模块 (L1/L2) | 竞品(SDS) 冗余菜单 | 映射类型与 AI 工作流重构解析 |
|---|---|---|
| **[L1] 全局数据大盘** | 【数字画像】 -> 学院/专业画像 | 画像不应是填报菜单。大模型定期扫描底层业务流数据，自动生成诊断摘要与报表。 |
| **[L1] 教学标准引擎** | 【系统管理】 -> 基础数据/部门/专业群 | 竞品将基础数据和表单建了十几个裸露的二级菜单。Yuanzhi 将其全部封装为"底层配置引擎"，对前端隐身。 |
| **[L1] 评价考核中枢** | 【评价考核管理】 -> 指标库 / 评价表 / 考核画像 / 评价任务 | 传统指标库需要一行行敲。Yuanzhi 引入大模型，直接将校级教学标准 PDF 拖入后台，AI 瞬间解析提取三级指标树。 |
| **[L1] 全局统筹与系统** | 【系统管理】 -> 权限/角色/日志/菜单配置 | 把所有权限分发、定时缓存清理、项目通用流程配置收口在最高权限的后台管控中。 |

| **[L1] 建设任务执行** | 【项目管理】 -> 项目申报 / 我的立项 | 将庞大项目库剥离。老师空间里只有"发给我的申报书卡片"和"我的微型任务"。 |
| **[L1] 教师档案袋** | 【成果库】 -> 论文/专利/教改等10类 | 彻底终结填表地狱。扔进 PDF 证明文件，多模态 AI 瞬间抽取核心字段，打上标签并计入积分！ |

### 1.3 代码架构与底层能力优化

1. **从单体架构到领域驱动微服务（DDD）**：拆分为 `context/course` (教学)、`context/program` (专业)、`context/plan` (培养方案)、`context/graph` (图谱引擎) 等清晰的限界上下文。
2. **多模态数据持久化体系**：**PostgreSQL (关系型) + PgVector (向量) + FalkorDB/Neo4j (图数据库)**混合存储。
3. **AI 原生工作流引擎**：审批流中穿插 AI 节点，实现人机协同审批。
4. **前端架构革新**：Vue 3 + Tailwind CSS，类 Notion/OS 的沉浸式多窗口交互。

## 第三章 基于 AI 赋能的用户故事与研发任务拆解

### 3.1 🏢 管理空间

**US-M01 全局数据大盘与 AI 数据洞察**
- 用户故事：校领导直观看到全校概览；通过自然语言向 Data Copilot 提问生成数据透视图表
- 研发任务：动态全局指标大屏、Text-to-SQL、后台定时巡检 Job
- 研发指引：前端参考 `HomeView.vue` + Echarts；后端复用 `AiChatService.java`

**US-M02 评价考核中枢极速生成**
- 用户故事：上传校级教学标准 PDF，AI 瞬间解析提取三级指标树
- 研发任务：PDF 解析、LLM Prompt 调优、前端指标树可视化
- 研发指引：前端树形控件参考 `TreeSelectWidget.vue`；后端复用 `ResourceExtractService.java`

**US-M03 敏捷的底层配置底座**
- 用户故事：系统管理员通过拖拽式设计器配置表单和审批流程
- 研发任务：FormRender 表单画布、BPMN 工作流引擎、UFO 数据同步
- 研发指引：前端复用 `admin_side_web/src/compoents/formRender`

### 3.2 🎯 专业空间

**US-P01 人培方案的 OBE 矩阵穿透**
- 用户故事：可视化 OBE 矩阵打通"毕业要求与课程体系"
- 研发任务：人培方案编辑器、CoPo 支撑度打分组件、AI 合规检查
- 研发指引：前端参考 `ProgramPlanView.vue`；后端复用 `CoPoMatrix` 实体

**US-P02 建设规划的长文自动裂变**
- 用户故事：点击"AI分解"按钮，长文自动裂变为可追踪的微任务卡片
- 研发任务：AI 任务提取器、任务卡片拆解派发、进度状态机
- 研发指引：后端参考 `PlanImportParser.java`；前端参考 `ProgramTasksView.vue`

**US-P03 教科研项目卡片化流水线**
- 用户故事：卡片式流水线管理项目；AI 自动提炼里程碑并扣减经费
- 研发任务：Kanban 看板、AI 文档摘要、经费台账
- 研发指引：前端借鉴 `ProgramProjectsView.vue` 引入拖拽库

### 3.3 👩‍🏫 教师空间

**US-T01 纵向 Agent 接管的四宫格工作台**
- 用户故事：收到单据后直接唤出侧边栏 AI Agent，帮我写好审核初稿并一键提交
- 研发任务：聚合待办列表、侧边栏 Copilot、函数调用封装
- 研发指引：前端复用 `GlobalDock.vue` + `AISidebar.vue`

**US-T02 课程标准的智能研制与图谱教案**
- 用户故事：输入核心知识点，AI 拉取人培矩阵生成课标；图谱引擎自动生成知识网
- 研发任务：课标生成 Prompt、GraphITI 引擎集成、知识图谱渲染
- 研发指引：后端调用 `graph_service/main.py` + `GraphApplicationService.java`；前端 `KnowledgeGraphView.vue`

**US-T03 多模态自动打标的教师档案袋**
- 用户故事：拖入 PDF/图片，多模态 AI 瞬间抽取字段并计入教研积分
- 研发任务：多文件拖拽上传、Qwen-VL/ GPT-4o OCR、积分映射计算
- 研发指引：前端复用 `FileCard.vue`；后端复用 `ResourceController.java` AI 分析接口
