### Q001：Data Copilot 的 Text-to-SQL 权限模型

- **优先级：** P0
- **分类：** security/data
- **状态：** pending
- **来源：** UC-M01 Step 4
- **背景：** 校领导通过自然语言向 Data Copilot 提问，后端调用 Text-to-SQL 将自然语言转为 SQL 查询数据库。若无权限控制，大模型可直接拼接任意 SQL，导致全校敏感数据泄露。
- **需要确认：**
  - Text-to-SQL 是否仅支持预置 SQL 模板，还是允许自由 SQL 生成？
  - 查询结果是否按当前用户角色数据范围过滤？
  - 是否需要查询审计日志？
- **推荐选项：**
  - A. 预置 SQL 模板（安全优先，推荐）
  - B. 自由 SQL + 角色数据范围过滤
  - C. 自由 SQL + 审计日志
  - D. 首版不做 Text-to-SQL，仅做固定报表
- **推荐答案：** A
- **默认假设：** 首版 Text-to-SQL 仅支持预设 SQL 模板，模板列表由系统管理员在后台配置，查询结果按当前用户角色数据范围过滤。
- **如果假设错误的影响：** 若实际需要自由 SQL，需新增权限网关、查询审计和防注入机制，预计增加 2 周开发周期。
- **澄清对象：** 产品经理 / 安全负责人

### Q002：AI 任务裂变的粒度与状态机

- **优先级：** P0
- **分类：** business/object
- **状态：** pending
- **来源：** UC-P02 Step 3
- **背景：** "长文自动裂变为可追踪的微任务卡片"——裂变的粒度决定任务卡片的 Schema 和状态机设计。不同粒度导致不同的实现方案。
- **需要确认：**
  - AI 按什么粒度裂变（语义段落/行动计划项/时间节点）？
  - 每张卡片需要包含哪些字段？
  - 任务状态机有几级？（待领取→进行中→已完成→已验收 是否够用？）
- **推荐选项：**
  - A. 按语义段落裂变，每段一张卡片（推荐）
  - B. 按行动计划项（动词开头）裂变，粒度更细
  - C. 按年度目标拆解为季度/月度里程碑
  - D. 首版先做手动任务创建，AI 拆分配套二期
- **推荐答案：** A
- **默认假设：** AI 按语义段落裂变，每段一张卡片；卡片状态机：待领取 → 进行中 → 已完成 → 已验收；卡片字段：任务描述 + 原文引用 + 负责人 + 截止日期 + 状态。
- **如果假设错误的影响：** 若粒度不同，卡片 Schema 和状态机需重设，影响前端渲染和后端状态管理。
- **澄清对象：** 产品经理 / 业务负责人

### Q003：表单拖拽配置器的配置边界

- **优先级：** P0
- **分类：** business/scope
- **状态：** pending
- **来源：** UC-M03 Step 2~7
- **背景：** "系统管理员通过拖拽式设计器配置表单和审批流程"——首版配置边界不明确，可能导致过度设计（工期翻倍）或配置不足（用户不认可）。
- **需要确认：**
  - 表单设计器首版支持哪些配置项？
  - 审批流程设计器首版支持单节点还是多节点？
  - 是否需要支持字段计算公式和外部数据源绑定？
- **推荐选项：**
  - A. 轻量配置：字段顺序+必填切换+单节点审批（推荐）
  - B. 中等配置：+A 基础上支持多节点审批和简单公式
  - C. 完整 BPMN 2.0 流程引擎
  - D. 首版不做设计器，提供固定表单模板
- **推荐答案：** A
- **默认假设：** 表单设计器首版仅支持：字段顺序调整、必填/非必填切换、字段显隐；审批设计器仅支持单节点审批（一个审批人）；不支持自定义字段类型、公式计算、外部数据源绑定。
- **如果假设错误的影响：** 若需要完整 BPMN，工期增加 3-8 周，建议二期迭代。
- **澄清对象：** 产品经理 / 技术负责人

### Q004：评审专家在专业空间的具体权限

- **优先级：** P0
- **分类：** permission
- **状态：** pending
- **来源：** 第二章角色权限映射表 / UC-P01 Step 7
- **背景：** 评审专家在专业空间有"核心视图"，但具体能做什么未定义。权限过宽可能修改正式数据，权限过窄可能无法完成评审职责。
- **需要确认：**
  - 评审专家是否只有只读权限，还是可以编辑/审批/驳回？
  - 评审意见是否需要区分"通过/不通过/有条件通过"？
  - 评审专家能否查看草稿状态的数据？
- **推荐选项：**
  - A. 只读（含历史版本）+ 提交评审意见（推荐）
  - B. 只读 + 可驳回（驳回后状态回退至草稿）
  - C. 只读 + 可审批通过（一键通过）
  - D. 首版不给评审专家开账号
- **推荐答案：** A
- **默认假设：** 评审专家在专业空间只有只读权限（含历史版本）和提交评审意见（文本）功能；不提供编辑/审批/驳回/删除权限。
- **如果假设错误的影响：** 若需要驳回功能，需新增：评审意见类型字段（通过/驳回）、驳回原因、状态驳回动作和驳回通知。
- **澄清对象：** 产品经理 / 业务负责人

- **优先级：** P1
- **分类：** business/ai
- **状态：** pending

- **优先级：** P1
- **分类：** business/rules
- **状态：** pending
- **来源：** UC-T03 Step 6

- **如果假设错误的影响：** 若实际需要积分兑换，需新增积分账户、兑换规则和事务机制。
- **澄清对象：** 产品经理 / 业务负责人

### Q009：多模态 OCR 自动打标的字段覆盖范围与文件限制

- **优先级：** P1
- **分类：** business/ai
- **状态：** pending
- **来源：** UC-T03 Step 3
- **背景：** "拖入 PDF/图片，多模态 AI 瞬间抽取字段"——支持哪些字段抽取？单文件大小/总文件数限制？是否支持批量？
- **默认假设：** 支持抽取字段：论文标题（title）、作者（author）、期刊名（journal_name）、发表年份（published_year）、DOI；单文件 ≤50MB；同一档案袋最多 50 个文件；批量上传串行处理，不支持并发。
- **如果假设错误的影响：** 若需要支持更多字段类型（如引用影响因子、期刊影响因子），需扩展 OCR Prompt 和模型微调。
- **澄清对象：** 产品经理 / AI 负责人

### Q010：AI 里程碑提炼的触发时机与判断标准

- **优先级：** P1
- **分类：** business/ai
- **状态：** pending
- **来源：** UC-P03 Step 5
- **背景：** "AI 自动提炼里程碑并扣减预算"——触发时机和判断标准不明确，可能导致误判。
- **默认假设：** 手动触发：用户上传中期汇报 PDF 后点击"AI 提炼里程碑"按钮；提炼结果展示给用户确认后扣减预算，不自动执行。
- **如果假设错误的影响：** 自动触发若无明确标准（如关键词/时间节点/金额阈值），可能产生误判，导致预算扣减错误。
- **澄清对象：** 产品经理 / 业务负责人

### Q011：跨空间数据权限边界

- **优先级：** P1
- **分类：** permission/data
- **状态：** pending
- **来源：** 第二章角色权限映射表
- **背景：** 系部主任/专业带头人在专业空间有核心视图，在管理空间有"协助执行"。协助执行具体能做什么？能否看到其他专业的数据？
- **默认假设：** 系部主任只能操作自己所属专业的数据；跨专业数据只有校领导和教务处可见；"协助执行"指在管理空间协助教务处管理员完成数据采集类操作，无独立配置权限。
- **如果假设错误的影响：** 若实际需要跨专业对比分析，需新增数据隔离配置和脱敏规则。
- **澄清对象：** 产品经理 / 安全负责人
