# 验收标准 / Acceptance Criteria

**文档：** Yuanzhi OS 竞品架构映射与 AI 重构解析
**版本：** v1.0
**日期：** 2026-06-09
**Gate 状态：** ⚠️ BLOCKED

---

> 验收标准使用 Given-When-Then 格式，按类别分组。
> 带 ⚠️ 的 AC 基于默认假设（assumed），待 P0/P1 澄清后更新。
> 带 🔒 的 AC 表示安全相关（涉及权限/数据/审计）。

---

## 分类一：正常主流程

### UC-M01：全局数据大盘自然语言查询

- **AC-M01-01：** Given 校领导已登录系统，When 输入自然语言查询（如"各专业学生人数"），Then 系统展示按专业分组的柱状图，数据加载时间 ≤ 3 秒
- **AC-M01-02：** Given 校领导已登录系统，When 输入未配置的自然语言查询，Then 系统提示"暂不支持该查询，请联系管理员配置"，不执行任何 SQL
- **AC-M01-03：** 🔒 Given SQL 执行成功，When 数据结果返回，Then 结果按当前用户数据范围过滤后展示，不泄露跨角色数据
- **AC-M01-04：** Given 数据图表已展示，When 校领导点击"导出"，Then 系统生成 Excel 或 PDF 格式的导出文件

### UC-M02：PDF 指标提取与三级指标树生成

- **AC-M02-01：** Given 教务处管理员上传标准格式 PDF，When AI 解析完成（≤ 30 秒），Then 指标树展示为三级树形结构（L1/L2/L3），每条指标显示名称和级别
- **AC-M02-02：** Given AI 解析完成，When 管理员点击"确认保存"，Then 指标树状态从草稿变为正式，计入考核评价，并收到保存成功提示
- **AC-M02-03：** ⚠️ Given AI 解析失败，When 系统提示后，Then 管理员可手动逐条新增指标，保存后转为正式状态
- **AC-M02-04：** Given 指标树处于草稿状态，When 管理员修改指标内容并保存，Then 修改被持久化，再次编辑时可看到修改内容
- **AC-M02-05：** Given AI 解析完成，When 指标项置信度 < 0.7，Then 该指标高亮标红，提示管理员人工核对

### UC-M03：表单拖拽设计与审批流程配置 ⚠️ ASSUMED（待 P0-003 澄清）

- **AC-M03-01：** Given 系统管理员在表单设计器中，When 拖拽字段到画布并保存，Then 字段出现在表单预览中，且字段顺序与设计一致
- **AC-M03-02：** Given 系统管理员设置字段为必填，When 用户提交表单时该字段为空，Then 系统阻止提交并高亮提示"该字段为必填项"
- **AC-M03-03：** ⚠️ Given 系统管理员配置单节点审批并发布，When 审批流程被触发，Then 系统通知审批人，审批人可通过或驳回
- **AC-M03-04：** ⚠️ Given 审批人被驳回，When 申请人查看到驳回原因，Then 申请内容退回到草稿状态，可重新编辑提交
- **AC-M03-05：** ⚠️ Given 系统管理员尝试配置多节点审批，When 点击保存，Then 系统提示"首版仅支持单节点审批，多节点审批将在二期支持"

### UC-P01：人培方案 OBE 矩阵研制

- **AC-P01-01：** Given 系部主任已创建人培方案，When 添加毕业要求并设置课程支撑度，Then OBE 矩阵可视化展示毕业要求与课程的支撑关系（支撑度分数 0~1）
- **AC-P01-02：** Given AI 检查合规性，When 课程支撑度覆盖率 < 80%，Then 系统展示警告"支撑度不足，建议补充课程"，但不阻止提交
- **AC-P01-03：** Given 系部主任提交评审，When 方案内容不完整（如缺少毕业要求或课程体系），Then 系统阻止提交并列出缺失项列表
- **AC-P01-04：** ⚠️ Given 评审专家已提交评审意见，When 系部主任查看，Then 看到评审意见内容和提交时间（assumed：评审专家只有意见提交权限）

### UC-P02：建设规划长文 AI 裂变为微任务卡片 ⚠️ ASSUMED（待 P0-002 澄清）

- **AC-P02-01：** ⚠️ Given 系部主任上传规划文档并点击"AI 分解"，When AI 裂变完成，Then 生成至少一张微任务卡片，每张卡片包含任务描述和原文引用（assumed：按语义段落裂变）
- **AC-P02-02：** ⚠️ Given 卡片列表已生成，When 系部主任为卡片分配负责人和截止日期，Then 该负责人收到任务通知
- **AC-P02-03：** Given 任务负责人领取任务，When 完成任务并标记完成，Then 状态从待领取→进行中→已完成
- **AC-P02-04：** Given 系部主任验收任务，When 任务状态=已完成，Then 状态变为已验收，完成后无法再修改状态
- **AC-P02-05：** Given AI 裂变失败，When 系统提示后，Then 管理员可手动创建任务卡片

### UC-P03：教科研项目 Kanban 看板管理

- **AC-P03-01：** Given 项目负责人上传中期汇报 PDF 并点击"AI 提炼里程碑"，When AI 提炼完成，Then 展示里程碑列表，包含时间节点、事项描述和预算影响金额
- **AC-P03-02：** ⚠️ Given 用户确认里程碑，When 扣减金额 ≤ 剩余预算，Then used_budget 更新，预算告警消失（assumed：手动触发+用户确认后扣减）
- **AC-P03-03：** Given 用户确认里程碑，When 扣减金额 > 剩余预算，Then 系统阻止扣减，提示"预算不足，无法扣减"
- **AC-P03-04：** Given 项目处于执行中，When 预算已用完，Then 项目卡片显示预算告警，但不影响继续执行

### UC-T01：四宫格工作台 AI 辅助审核

- **AC-T01-01：** Given 教师有待处理单据，When 点击单据，Then AI Agent 侧边栏打开，展示单据摘要
- **AC-T01-02：** Given AI Agent 已生成审核意见初稿，When 教师提交，Then 审核意见写入数据库，单据从待办移至已完成
- **AC-T01-03：** Given AI Agent 生成失败，When 教师手动填写审核意见并提交，Then 功能降级可用，流程正常完成

### UC-T02：课程标准 AI 研制与知识图谱生成 ⚠️ ASSUMED（待 P1-003/P1-004 澄清）

- **AC-T02-01：** ⚠️ Given 教师输入核心知识点并点击"AI 生成课标"，When AI 生成完成，Then 课标内容填充，状态为草稿（assumed：AI 生成内容需人工确认后转正式）
- **AC-T02-02：** Given 课标处于草稿状态，When 教师点击"提交评审"，Then 状态变为评审中，评审专家收到通知
- **AC-T02-03：** Given 课标已发布，When 教师点击"生成知识图谱"，Then GraphITI 开始生成，状态变为生成中
- **AC-T02-04：** ⚠️ Given GraphITI 生成完成，When 教师确认发布，Then 知识图谱状态变为已发布，供查阅（assumed：课程级手动触发）

### UC-T03：教师档案袋多模态 OCR 自动打标 ⚠️ ASSUMED（待 P1-002/P1-005 澄清）

- **AC-T03-01：** ⚠️ Given 教师上传 PDF 文件，When AI OCR 抽取完成，Then 展示抽取字段列表（标题/作者/期刊名/年份/DOI），每个字段显示置信度（assumed：固定字段集）
- **AC-T03-02：** ⚠️ Given AI 抽取完成，When 字段置信度 < 0.7，Then 该字段高亮标红，提示人工核对（assumed：置信度阈值 0.7）
- **AC-T03-03：** ⚠️ Given 教师点击"确认并计入积分"，When 积分计算完成，Then 积分账户总计增加对应积分，档案袋展示积分明细（assumed：预置积分规则）
- **AC-T03-04：** Given 教师上传非标准格式文档，When AI 抽取失败，Then 系统提示"无法识别，请手动输入"，教师可手动填写字段

## 分类二：角色与权限 🔒

- **AC-PERM-01：** 🔒 Given 用户角色=校领导，When 访问管理空间，Then 仅显示核心视图，无编辑/配置/审批权限
- **AC-PERM-02：** 🔒 Given 用户角色=系部主任，When 访问其他专业数据，Then 系统拒绝访问，返回"无权查看该数据"
- **AC-PERM-03：** 🔒 Given 用户角色=评审专家，When 尝试编辑专业空间数据，Then 系统拒绝操作，提示"您只有只读权限"
- **AC-PERM-04：** 🔒 Given 用户角色=系统管理员，When 尝试访问业务数据（如人培方案/项目），Then 系统拒绝访问，仅可访问配置数据
- **AC-PERM-05：** 🔒 Given 用户角色=一线教师，When 上传文件大小 > 50MB，Then 系统拒绝上传，提示"单文件大小不得超过 50MB"
- **AC-PERM-06：** 🔒 Given Text-to-SQL 查询，When SQL 涉及用户无权查看的数据，Then 结果按数据范围过滤后返回，不返回原始 SQL 结果

## 分类三：状态流转

- **AC-STATE-01：** Given 三级指标树状态=待提取，When AI 解析完成，Then 状态自动变为草稿，无需人工干预
- **AC-STATE-02：** Given 三级指标树状态=草稿，When 管理员点击"确认保存"，Then 状态变为正式，可用于考核
- **AC-STATE-03：** ⚠️ Given 微任务卡片状态=待领取，When 负责人领取任务，Then 状态变为进行中，时间戳更新
- **AC-STATE-04：** Given 微任务卡片状态=进行中，When 负责人标记完成，Then 状态变为已完成
- **AC-STATE-05：** Given 微任务卡片状态=已完成，When 系部主任验收，Then 状态变为已验收，时间戳更新
- **AC-STATE-06：** ⚠️ Given 成果记录状态=提取中，When AI OCR 抽取完成，Then 状态变为已确认（自动）或保留待提取（失败）（assumed）
- **AC-STATE-07：** Given 教科研项目状态=申报中，When 教务处审批通过，Then 状态变为已立项，自动进入 Kanban 执行中列
- **AC-STATE-08：** Given 教科研项目状态=执行中，When 上传结项材料并审批通过，Then 状态变为已结项，填充 ended_at

## 分类四：异常与边界

- **AC-ERR-01：** Given PDF 上传，When 文件格式不是 PDF 或大小超过 100MB，Then 系统拒绝上传并提示"仅支持 PDF 文件，且大小不超过 100MB"
- **AC-ERR-02：** Given AI 服务不可用，When 用户触发任何 AI 功能，Then 系统提示"AI 服务暂时不可用，请稍后重试"，功能降级可用（手动模式）
- **AC-ERR-03：** Given 用户提交表单，When 必填字段为空，Then 系统阻止提交并高亮提示缺失字段，不发送任何请求
- **AC-ERR-04：** Given 重复提交，When 后端检测到重复提交（相同表单+相同用户+时间间隔<30s），Then 返回"该表单已提交，请勿重复操作"
- **AC-ERR-05：** Given 任务负责人超期未完成任务（超截止日期7天），When 系统检测到超期，Then 通知系部主任"任务[xxx]已超期7天，请重新分配"
- **AC-ERR-06：** Given 预算扣减，When 扣减后 used_budget > total_budget，Then 系统阻止扣减，提示"预算不足，无法完成扣减操作"

## 分类五：数据、审计、通知

- **AC-AUDIT-01：** 🔒 Given 用户执行任何创建/编辑/删除/提交操作，When 操作完成，Then 系统写入审计日志（日志内容：用户ID/操作类型/对象ID/时间戳/变更前后内容）
- **AC-AUDIT-02：** 🔒 Given 审计日志，When 任何用户尝试删除或修改日志，Then 系统拒绝，日志仅追加不可修改
- **AC-NOTIFY-01：** Given 教务处管理员上传 PDF 成功，When AI 解析完成，Then 系统通知上传用户"您的教学标准已解析完成，请前往确认"
- **AC-NOTIFY-02：** Given 系部主任为卡片分配负责人，When 分配完成，Then 负责人收到通知"您被分配了任务[xxx]，请及时处理"
- **AC-NOTIFY-03：** Given 项目负责人上传中期汇报，When AI 提炼完成，Then 负责人收到通知"您的项目里程碑已提炼完成，请确认扣减预算"
- **AC-NOTIFY-04：** Given 评审专家提交评审意见，When 提交完成，Then 系部主任收到通知"评审专家已提交意见，请查看"

## 分类六：AI 行为与人工兜底 ⚠️

- **AC-AI-01：** ⚠️ Given Text-to-SQL 查询，When 用户输入未配置的自然语言，Then 系统返回"暂不支持该查询"，不执行任何 SQL（assumed：仅预置模板）
- **AC-AI-02：** ⚠️ Given AI 任务裂变，When 裂变失败，Then 系统提示"AI 分解失败，请手动创建任务卡片"（assumed：粒度按语义段落）
- **AC-AI-03：** ⚠️ Given AI 课标生成，When AI 服务返回错误，Then 课标内容保留草稿状态，教师可手动填写（assumed：草稿→人工确认）
- **AC-AI-04：** ⚠️ Given GraphITI 图谱生成，When 生成失败，Then 教师收到通知"图谱生成失败，请稍后重试"，不改变原图谱状态（assumed：课程级手动触发）
- **AC-AI-05：** ⚠️ Given 多模态 OCR，When 识别置信度 < 0.7，Then 字段高亮标红，提示人工核对（assumed：置信度阈值 0.7）
