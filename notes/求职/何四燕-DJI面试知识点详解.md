# 一、产品经理核心知识

## 1.1 用户需求挖掘

### 常用方法

| 方法 | 适用场景 | 操作要点 |
|---|---|---|
| 用户访谈 | 深度理解需求 | 5-10人，开放式问题，避免引导 |
| 问卷调查 | 大样本验证 | 问卷设计避免诱导性措辞 |
| 数据分析 | 验证假设 | 埋点数据、转化漏斗、用户路径 |
| 竞品分析 | 行业对标 | 功能对比、优劣势、使用体验 |
| 可用性测试 | 验证方案 | 原型测试、任务完成率 |

### 需求分析模型

**Kano模型**
- **基本需求**：没有用户会不满，但有也不会特别满意
- **期望需求**：越满足越满意（线性关系）
- **兴奋型需求**：超出预期，让用户惊喜

**需求优先级方法**

| 方法 | 公式 | 适用场景 |
|---|---|---|
| RICE | (Impact × Confidence × Reach) / Effort | 多需求横向比较 |
| MoSCoW | Must/Should/Could/Won't | 资源有限时的优先级 |
| ICE | Impact × Confidence × Ease | 快速评分 |

## 1.2 产品规划与Roadmap

### Roadmap制定步骤

1. **产品愿景**：长期目标（1-3年）
2. **战略分解**：年度目标、季度目标
3. **需求池管理**：收集→筛选→排序→进入迭代
4. **版本规划**：功能分组、时间节点、里程碑
5. **动态调整**：根据数据和反馈迭代

### Roadmap展示方式

- **时间轴型**：按月/季度规划
- **主题型**：按产品主题（而非单个功能）组织
- **目标导向型**：以业务目标为维度

---

## 1.3 PRD写作规范

### PRD核心要素

1. **背景与目标**：为什么做？要达成什么？
2. **用户与场景**：谁用？什么场景？
3. **功能需求**：做什么？（User Story格式）
4. **非功能需求**：性能、安全、兼容性
5. **验收标准**：怎么做算成功？
6. **运营计划**：上线后如何运营？

### User Story格式

```
作为 [用户角色]
我想要 [功能]
以便 [实现价值]
```

## 1.4 数据埋点设计

### 核心概念

| 术语 | 含义 |
|---|---|
| event | 用户行为（如button_click） |
| property | 事件属性（如button_name） |
| user_id | 用户唯一标识 |
| pageview | 页面访问 |
| funnel | 转化漏斗 |

### 埋点原则

1. **完整性**：关键路径全覆盖
2. **准确性**：事件命名规范、属性一致
3. **可追溯**：能还原用户操作路径
4. **避免过度**：埋点也是成本

### 关键指标

- **DAU/MAU**：日活、月活
- **留存率**：次日留存、7日留存
- **转化率**：每步漏斗转化
- **NPS**：用户净推荐值

---

## 1.5 A/B测试

### 步骤

1. **提出假设**：优化方向
2. **设计实验**：流量分配、实验周期
3. **实施**：开发、发布
4. **分析**：统计显著性
5. **决策**：数据说话

### 统计概念

- **统计显著性**：p-value < 0.05
- **最小样本量**：根据基线转化率和最小可检测差异计算
- **置信区间**：结果的可信范围

### 常见错误

- 测试周期太短
- 样本不均匀
- 多重比较问题

# 二、AI/大模型知识

## 2.1 AI基础概念

### 什么是人工智能

**AI（Artificial Intelligence）**：让机器具有人类智能的技术

**关键分支**：
- ML（Machine Learning）：从数据中学习
- DL（Deep Learning）：神经网络
- NLP（Natural Language Processing）：语言理解
- CV（Computer Vision）：图像识别
- 多模态：融合多种感知能力

### 机器学习分类

| 类型 | 特点 | 应用场景 |
|---|---|---|
| 监督学习 | 有标签数据 | 分类、回归 |
| 无监督学习 | 无标签 | 聚类、异常检测 |
| 强化学习 | 通过奖励学习 | 游戏、推荐 |

---

## 2.2 大语言模型（LLM）

### 什么是LLM

**大语言模型**：基于Transformer架构、训练于大规模文本的模型

**核心能力**：
- 文本生成
- 问答对话
- 代码生成
- 逻辑推理
- 知识总结

### 主流LLM

| 模型 | 公司 | 特点 |
|---|---|---|
| GPT-4 | OpenAI | 最强推理能力 |
| Claude | Anthropic | 长文本、风格控制 |
| Gemini | Google | 多模态原生 |
| 通义千问 | 阿里 | 中文优化 |
| 文心一言 | 百度 | 中文优化 |
| Kimi | 月之暗面 | 长上下文 |

### LLM局限

- **幻觉**：一本正经地胡说八道
- **时效性**：知识有截止日期
- **算力成本**：推理成本高
- **黑盒**：可解释性差

---

## 2.3 多模态模型

### 什么是多模态

**多模态**：同时处理文本、图像、视频、音频等多种数据类型的AI模型

### 多模态能力

| 能力 | 说明 | 示例 |
|---|---|---|
| 图文理解 | 理解图像内容 | 看图回答问题 |
| 图文生成 | 图像描述生成 | 图像 caption |
| 视觉问答 | 基于图像对话 | "图里有什么？" |
| 视频理解 | 分析视频内容 | 视频摘要 |

### 核心模型

- **CLIP**（OpenAI）：图文对比学习
- **GPT-4V**：视觉语言模型
- **Gemini**：原生多模态
- **LLaVA**：开源视觉助手

### 无人机应用场景

1. **智能拍摄**：场景识别（风景/建筑/人像）
2. **环境感知**：实时画面分析、障碍物检测
3. **语音交互**：语音控制+视觉识别
4. **航拍剪辑**：素材理解+自动生成

## 2.4 RAG架构

### 什么是RAG

**RAG（Retrieval-Augmented Generation）**：检索增强生成

**解决的问题**：LLM知识过时+幻觉问题

### RAG流程

```
用户问题
    ↓
Query处理（向量化）
    ↓
向量检索（Vector Search）
    ↓
相关文档检索
    ↓
Context构建（问题+检索内容）
    ↓
LLM生成回答
    ↓
回答输出
```

### 核心组件

| 组件 | 作用 | 技术选型 |
|---|---|---|
| 文档加载 | 读取各种格式 | PDF解析、HTML解析 |
| 分块 | 文档切分 | Text splitter |
| Embedding | 文本转向量 | BGE、M3E、OpenAI |
| 向量数据库 | 存储+检索 | Milvus、Pinecone |
| LLM | 生成回答 | GPT-4、Claude |

### Chunking策略

- **固定大小**：500/1000 tokens
- **语义分割**：按段落/章节
- **混合策略**：固定大小 + 重叠

### Embedding模型

| 模型 | 说明 |
|---|---|
| OpenAI text-embedding-3 | 最强，但收费 |
| BGE | 国产开源，中文优化 |
| M3E | 国产开源，轻量 |

## 2.5 Agent架构

### 什么是Agent

**Agent**：能自主决策、调用工具、完成复杂任务的AI系统

### Agent核心组件

1. **规划（Planning）**：任务分解、反思
2. **记忆（Memory）**：短期+长期记忆
3. **工具（Tools）**：调用外部能力
4. **行动（Action）**：执行决策

### Agent框架

| 框架 | 特点 |
|---|---|
| LangChain | 最流行，生态丰富 |
| LlamaIndex | 知识检索优化 |
| AutoGen | 多Agent协作 |
| CrewAI | 多Agent角色扮演 |

### 产品经理视角的Agent应用

1. **智能客服**：意图识别→知识检索→回答
2. **个人助手**：任务拆解→工具调用→结果整合
3. **数据分析**：理解问题→写代码→执行→可视化

## 2.6 模型部署

### 端云协同

| 方案 | 优点 | 缺点 |

# 五、数据分析知识

## 5.1 常用指标

### 用户指标

| 指标 | 公式/说明 |
|---|---|
| DAU | 日活跃用户数 |
| MAU | 月活跃用户数 |
| DAU/MAU | 活跃度（越高越好） |

### 业务指标

| 指标 | 说明 |
|---|---|
| 转化率 | 目标完成/访问量 |
| 跳出率 | 仅访问一页的比例 |
| 平均订单价值 | 总价值/订单数 |

### 漏斗分析

```
访问 → 注册 → 首次使用 → 活跃 → 付费
  ↓       ↓        ↓        ↓
各环节转化率 → 识别瓶颈 → 优化策略
```

## 5.3 SQL基础（产品经理需要了解）

```sql

-- 统计DAU
SELECT date, COUNT(DISTINCT user_id)
FROM events
GROUP BY date

  COUNT(DISTINCT user_id) as users,
```

# 六、面试技巧

## 6.1 回答框架

### STAR法则

| 要素 | 内容 |
|---|---|
| Situation | 背景/情境 |
| Task | 任务/挑战 |
| Action | 采取的行动 |
| Result | 结果/成果 |

### 产品经理版STAR

1. **背景**：项目的上下文
2. **用户**：谁遇到了什么问题
3. **方案**：我设计了什么
4. **结果**：上线后的数据效果

---

## 6.2 常见陷阱

### ❌ 避免的回答

1. **只讲团队配合**：重点是我做了什么
2. **没有数据**：每个成就要有量化指标
3. **过度谦虚**：适当展示能力，但不要吹牛
4. **只讲技术**：产品经理面试，技术是辅助

### ✅ 好的回答

1. **用户导向**：从用户痛点出发
2. **结果导向**：用数据说话
3. **思考过程**：展示分析问题的方法
4. **反思复盘**：项目中的教训

---

## 6.3 反问环节

### 高质量问题

1. **团队结构**：产品团队多少人？分工如何？
2. **产品方向**：这个职位主要负责哪条产品线？
3. **技术栈**：团队用什么技术？AI应用到什么程度？
4. **成长空间**：这个岗位的成长路径是什么？
5. **面试流程**：后续面试安排？

### 避免问的问题

- 薪资（第一轮不问）
- 加班（第一轮不问）
- 福利（等HR主动说）
